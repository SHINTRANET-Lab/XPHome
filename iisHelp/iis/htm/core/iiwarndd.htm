ASP のデバッグを有効にする

Microsoft Script Debugger を使用すると、ASP スクリプトのエラーを見つけることができます。Web サーバーでデバッガを使用するには、まずここで説明する手順に従ってサーバーをデバッグ用に構成する必要があります。デバッガを使ってスクリプトを調べる方法については、「ASP スクリプトをデバッグする」および Microsoft Scripting Technologies ホーム ページを参照してください。

ASP のデバッグを有効にするには
  1. IIS スナップインで、Web サイトまたはアプリケーションの開始点ディレクトリを選択します。
  2. ディレクトリのプロパティ シートを開き、[ホーム ディレクトリ] タブ、[仮想ディレクトリ] タブ、または [ディレクトリ] タブをクリックします。
  3. [構成] をクリックし、[アプリケーションのデバッグ] タブをクリックします。
  4. デバッグを有効にするには、[ASP のサーバー側のスクリプトのデバッグを有効にする] チェック ボックスをオンにします。スクリプトにエラーがある場合、または ASP がスクリプト内のブレークポイントに到達すると、デバッガが起動します。
