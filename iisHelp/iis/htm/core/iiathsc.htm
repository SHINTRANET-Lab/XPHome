認証

ユーザーにサーバーとのネットワーク接続を許可する前に "認証"、つまりユーザーの Windows ユーザー アカウントの識別を行うよう、IIS を構成できます。ただし、ユーザーの認証が行われるのは、匿名アクセスが無効になっている場合、または NTFS のアクセス権の設定により、ユーザーが有効な Windows ユーザー アカウントのユーザー名およびパスワードで身元を証明する必要がある場合に限られます。

IIS によって提供される認証オプションを使用すると、セキュリティに必要な条件およびユーザーの Web ブラウザの機能に合った認証方法を選択することができます。

ここでは、次の項目について説明します。

  • 「認証について」では、認証の概要について説明します。
  • 「認証を有効にし、構成する」では、基本認証、ダイジェスト認証、および統合 Windows 認証を有効にし、構成する手順について説明します。
  • 「既定のログオン ドメインを設定する」では、基本認証のログオン ドメインを構成する手順について説明します。
  • 「IIS で Fortezza セキュリティを使用する」では、IIS の Fortezza セキュリティ機能を構成する手順について説明します。
