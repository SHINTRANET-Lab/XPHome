[Active Server Pages] プロパティ シート

このプロパティ シートは、選択したアプリケーションで、ASP スクリプトをどのように実行するかを設定するときに使用します。

[セッションの状態を有効にする] チェック ボックス

セッションの状態を有効または無効にします。

セッションの状態を有効にすると、Active Server Pages は ASP アプリケーションにアクセスするユーザーごとにセッションを作成するので、アプリケーションのページ全体でユーザーを識別できるようになります。セッションの状態を無効にすると、ASP はユーザーを追跡しなくなり、ASP スクリプトは Session オブジェクトに情報を格納したり、Session_OnStart イベントや Session_OnEnd イベントを使用したりできなくなります。ユーザーがタイムアウト時間の終了までにアプリケーションのページの要求や更新を行わなかった場合、セッションは自動的に終了します。

タイムアウト時間を変更するには、[セッションのタイムアウト] ボックスに新しい数字を入力します。

スクリプトは、Session.Abandon メソッドを使用して、セッションを明示的に終了できます。アプリケーションのセッションの状態が有効になっている場合でも、<%@ ENABLESESSIONSTATE = False %> ディレクティブを使って個別の ASP ページのセッションの状態を無効にすることができます。

[バッファ処理を有効にする] チェック ボックス

ブラウザへの出力をバッファ処理するには、このチェック ボックスをオンにします。

このチェック ボックスをオンにすると、ASP ページが生成するすべての出力が、ブラウザへ送られる前に収集されます。チェック ボックスをオフにすると、出力はページが処理されるときにブラウザに返されます。バッファ処理を有効にすると、ASP スクリプトのどこからでも HTTP ヘッダーを設定できるようになります。スクリプトの中でこのオプションを無効にするには、Response.Buffer メソッドを使用します。

[親のパスを有効にする] チェック ボックス

ASP スクリプトで、現在のディレクトリの親ディレクトリへの相対パス (.. 構文を使ったパス) を使用するには、このチェック ボックスをオンにします。

このオプションを有効にする場合は、親ディレクトリに実行アクセス権を設定しないでください。実行アクセス権を設定すると、スクリプトが親ディレクトリで許可されていないプログラムの実行を試みる場合があります。

[既定の ASP の言語] ボックス

Active Server Pages の主要スクリプト言語を指定します。この言語は、ASP デリミタ (<% および %>) 内のコマンド処理に使用されます。

選択したアプリケーションのすべてのページに別の主要スクリプト言語を選択するには、テキスト ボックスにその言語名を入力します。

ASP には、Microsoft Visual Basic Scripting Edition (VBScript) と Microsoft JScript (JScript) の 2 種類の ActiveX スクリプト エンジンが添付されています。[既定の ASP の言語] の初期値は、vbscript です。サーバーにインストールされている ActiveX スクリプト エンジンに対応する言語であれば、どの言語名でも指定できます。スクリプト エンジンの製造元が指定する、エンジンに必要な正確なキーワードを使用してください。ASP ページで既定の言語を無効にするには、<%@ LANGUAGE %> ディレクティブを使用します。

[スクリプトのタイムアウト] ボックス

ASP がスクリプトの実行を許可する時間の長さを指定します。

タイムアウト時間内にスクリプトの実行が終了しなかった場合、ASP はスクリプトを停止し、Windows イベント ログにイベントを記録します。タイムアウト時間には、1 から 2147483647 までの値を指定できます。ASP スクリプトの中でこのオプションを無効にするには、Server.ScriptTimeout メソッドを使用します。

注   [アプリケーションの設定] のプロパティは、Web サイト、仮想ディレクトリ、およびディレクトリの各レベルで設定できます。
