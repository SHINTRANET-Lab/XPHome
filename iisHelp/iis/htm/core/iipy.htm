[Web サイト] プロパティ シート

このプロパティ シートは、Web サイトの識別パラメータを設定するために使用します。サイトの IP アドレスを設定するには、管理しているコンピュータ上で、あらかじめそのサイトの TCP/IP の設定を構成しておく必要があります。

[識別]

説明

任意のサーバー名を入力します。この名前は、インターネット サービス マネージャ (HTML) のツリー ビューに表示されます。

[詳細] をクリックして、IP アドレス、TCP ポート番号、SSL ポート番号、およびホスト ヘッダー名を構成します。

[IP アドレス]

このボックスにアドレスを表示するには、管理しているコンピュータで使用するアドレスを、あらかじめコントロール パネルで定義しておく必要があります。詳細については、Windows のマニュアルを参照してください。特定の IP アドレスを割り当てない場合、このコンピュータに割り当てられていて、ほかのサイトに割り当てられていないすべての IP アドレスにこのサイトが応答し、既定の Web サイトになります。

[TCP ポート]

サービスを実行するポートを指定します。既定値はポート 80 です。ポートの値は、任意の重複しない TCP ポート番号に変更できますが、あらかじめクライアントにそのポート番号を通知しておく必要があります。通知せずに変更すると、クライアントはサーバーに接続できなくなります。ポート番号は必ず指定します。

接続

[無制限]

同時接続を無制限に許可する場合は、このオプションを選択します。

[最大接続数]

サイトへの同時接続数を制限する場合は、このオプションを選択します。許可する最大接続数をボックスに入力します。

[接続のタイムアウト]

サーバーが、反応のないユーザーとの接続を切断するまでの時間を秒単位で設定します。これにより、HTTP プロトコルが接続の解除に失敗しても、すべての接続が切断されます。

[HTTP キープ アライブを有効にする]

この機能によりクライアントは、要求のたびに接続を行う必要がなくなり、クライアントとサーバーとの接続を維持できます。キープアライブを無効にすると、サーバーのパフォーマンスが低下する場合があります。キープアライブは、既定では有効に設定されています。

[ログを収集する]

このチェック ボックスをオンにすると、Web サイトのログ収集機能が有効になります。これにより、ユーザーの利用状況を詳細に記録し、希望の形式でログを作成できます。ログの収集を有効にし、[アクティブ ログ形式] ボックスでログの形式を選択します。選択できる形式は、次のとおりです。

  • [Microsoft IIS ログ ファイル形式] 固定の ASCII 形式です。
  • [NCSA 共通ログ ファイル形式] National Center for Supercomputing Applications (NCSA) の共通形式です。固定の ASCII 形式です。
  • [W3C 拡張ログ ファイル形式] カスタマイズ可能な ASCII 形式です。既定で選択されています。プロセス総計を使用する場合は、必ずこの形式を選択します。
  • [ODBC ログ] (Windows 2000 Server でのみ使用可能) データベースに記録される固定の形式です。

ログ ファイルの作成条件 (毎週、ファイル サイズ別など) を構成する場合、または W3C 拡張形式や ODBC 形式のログ収集のプロパティを構成する場合は、[プロパティ] をクリックし、設定します。
