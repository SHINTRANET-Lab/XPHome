Common Gateway Interface (CGI)
ソフトウェア サービスを起動するためのサーバー側インターフェイス。HTTP サービスのような情報サービスと、データベースやほかのプログラムなどの、サーバーのホスト コンピュータ上にあるリソースとの間で通信する方法を規定した仕様です。たとえば、ユーザーが Web ブラウザからフォームを送信すると、HTTP サービスは CGI スクリプトと呼ばれるプログラムを実行し、CGI を通じてそのプログラムにユーザーの入力情報を渡します。 プログラムは、CGI を通じて HTTP サービスに情報を返します。CGI 標準に準拠した入出力が行われていれば、どのようなソフトウェアでも CGI プログラムとして機能できます。CGI アプリケーションは、常にプロセス外で実行されます。「サーバー」も参照してください。

Common Object Request Broker Architecture (CORBA)
Object Management Group が 1992 年に規定した仕様。この仕様では、2 つのプログラムが、異なるプログラミング言語で作成され、異なるプラットフォームで実行されている場合でも、一方のプログラムを構成するオブジェクトが、もう一方のプログラムのオブジェクトと通信することができます。 プログラムは、オブジェクト リクエスト ブローカ (ORB) を通じてオブジェクトを要求するため、そのオブジェクトが属するプログラムの構造を知る必要がありません。CORBA は、オブジェクト指向環境で機能するように設計されています。

Windows の場合には、実行可能プログラム、仮想メモリのアドレス群、およびスレッドで構成されるオブジェクト。UNIX の場合は、スレッドと同じ意味。「スレッド」も参照してください。

プロセス総計
CGI スクリプトとアウトプロセス アプリケーションのリソース消費を、管理者が監視してログ収集できるようにする IIS の機能。

プロトコル
インターネット上でコンピュータが通信するための手段。HTTP は、World Wide Web で最も一般的なプロトコルです。ほかに FTP、Gopher、telnet などのインターネット プロトコルがあります。プロトコルは、リソースの完全な URL の一部を構成します。

分散処理
通信ネットワークを介して接続された各コンピュータで作業を処理する情報処理形式。分散処理は通常、単純な分散処理か真の分散処理のいずれかに分類されます。単純な分散処理は、互いに通信可能なコンピュータ間で作業負荷を共有します。 真の分散処理では、各コンピュータが異なるタスクを行い、最終的にタスクを統合することによって大きいタスクを実行します。真の分散処理には、ハードウェアとソフトウェアが通信し、リソースを共有し、自由に情報を交換するための高度な構造化環境が必要です。

分離開発
アプリケーション開発者が Web サーバーに接続せずにスクリプトを作成およびテストできるようにする機能。いくつかのソフトウェアでこの機能が提供されています。

ホーム ページ

ホスト名のフィルタ処理

マーシャリング

マルチ アイデンティティ

マルチホーム ホスト

ミドルウェア
アプリケーション、オペレーティング システム、およびネットワーク層とトランスポート層の間に位置するネットワーク対応のシステム ソフトウェア。ミドルウェアの目的は、連携処理の一部を容易にすることです。ディレクトリ サービス、メッセージ受け渡しメカニズム、分散トランザクション処理 (TP) モニタ、オブジェクト リクエスト ブローカ、リモート プロシージャ コール (RPC) サービス、データベース ゲートウェイなどは、ミドルウェアの例です。

ミドル層
アプリケーション サーバー層とも呼ばれます。データベースと、ユーザー インターフェイスまたは Web クライアント間の論理層です。通常、Web サーバーが常駐する層であり、ビジネス オブジェクトがインスタンス化される層でもあります。「データ ソース層」も参照してください。

ミラー セット
データ全体の冗長なコピー、つまりシャドー コピー。ミラー セットは、選択したディスクと完全に一致するコピーを提供します。元のディスクに書き込まれたすべてのデータは、シャドー (ミラー) ディスクにも書き込まれます。元のディスクで障害が発生した場合、ユーザーは同じ情報のコピーがある別のディスクに直ちにアクセスすることができます。ミラー セットは、フォールト トレランスを提供します。「フォールト トレランス」も参照してください。

メイン スレッド
"シングル スレッド対応" としてマークされているコンポーネントのオブジェクトすべてを実行するために使用される単一スレッド。「アパートメント スレッド」も参照してください。

呼び出し
プログラムの実行を、コードのほかの部分に移すこと。普通は、サブルーチンに処理を移します。呼び出された部分の処理が完了したとき、呼び出した場所からプログラムの実行を再開できるように、必要な情報が保存されます。通常、呼び出されたサブルーチンには、引数やパラメータと呼ばれる 1 つまたは複数の値が渡されます。サブルーチンはこれらの値を使用し、必要に応じて変更します。

呼び出し元
オブジェクトのメソッドを呼び出すクライアント。オブジェクトの呼び出し元は、オブジェクトの作成元とは限りません。たとえば、クライアント A がオブジェクト X を作成し、その参照をクライアント B に渡し、クライアント B がその参照を通じてオブジェクト X のメソッドを呼び出す場合があります。この場合は、クライアント A が作成元であり、クライアント B が呼び出し元です。

ルーター
メッセージを配信する通信ネットワーク上の中継装置。多数のコンピュータが網目状に接続された単一のネットワークでは、ルーターは伝送されたメッセージを受信し、それらを最も効率的な経路で正しい宛先に送信します。同じ通信プロトコルを使用する相互接続されたローカル エリア ネットワーク (LAN) 間では、ルーターは LAN 間のリンクとして機能し、メッセージを相互に送信できるようにします。

レジストリ
Windows の中央階層データベース。単一または複数のユーザー、アプリケーション、およびハードウェア デバイスのシステム構成に必要な情報の格納に使用されます。レジストリには、システム動作中に常に参照される情報が格納されます。この情報には、各ユーザーのプロファイル、コンピュータ中のインストール済みアプリケーション、作成可能なドキュメントのタイプ、フォルダやアプリケーション アイコンのプロパティ シートの設定、システムに組み込まれているハードウェア、使用中のポートなどが含まれます。
