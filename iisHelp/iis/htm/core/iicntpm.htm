カウンタ リファレンス

カウンタ データを参照するには、パフォーマンス モニタ、サードパーティ製のクライアント ソフトウェア、またはパフォーマンス カウンタへのアクセスを可能にする 2 種類のプロトコルのどちらかを使用しているカスタム アプリケーションが使用されます。2 種類のプロトコルとは、Simple Network Management Protocol (SNMP) と Performance Data Helper (PDH) です。次の表で比較を行います。プロトコルのヘルパ ファイルに対しては、編集が許可されません。独自のカスタム アプリケーションを作成してパフォーマンス カウンタにアクセスする場合は、ヘルパ ファイルを参照できます。両プロトコルのインターフェイスについては、MSDN Online library を参照してください。

プロトコル PDH SNMP
使用対象 パフォーマンス モニタ (またはカスタム アプリケーション)。 MIB ブラウザとも呼ばれる、サードパーティ製クライアント ソフトウェア (またはカスタム アプリケーション)。
セットアップ 既定でインストールされる。 Windows のインストール段階または [プログラムの追加と削除] で、[管理とモニタ ツール] から選択する。

ヘルパ ファイルの内容 識別名、カウンタ名、説明。 識別名、カウンタ名、説明、一意のオブジェクト ID 番号 (OID)、アクセス方法や状態などその他の属性。
長所 プログラミング インターフェイスが容易。Microsoft 製のソフトウェアで使用されるため、安定しており、しかも強固。WMI からアクセス可能。 ほかのオペレーティング システムのユーザーには馴染み深い。WMI からアクセス可能。IIS カウンタは読み取り専用だが、SNMP は管理プロトコルであるため、値の書き込みが可能な場合もある。
短所 PDH は管理プロトコルではなく監視プロトコルなので、値の書き込みは不可。 ヘルパ ファイルでは順序と OID が非常に重要なので、カウンタの追加が困難。

インターネット インフォメーション サービスと同時にインストールされる Web Service オブジェクト カウンタと FTP Service オブジェクト カウンタは、ファイル転送率、帯域幅の使用状況、接続率、エラー、およびユーザーの数と種類を監視します。オブジェクトのインスタンス全体 (Total インスタンスを使用)、または [既定の Web/FTP サイト] や [管理者 Web サイト] など特定のインスタンスについて、パフォマンス データを表示できます。 IIS Global オブジェクト カウンタは、すべての IIS サービスについてパフォーマンス データを通知します。Active Server Pages オブジェクト カウンタは、実行時エラー、要求、セッション、テンプレート、およびトランザクションを含むスクリプトの接続状況を監視します。

次に、IIS パフォーマンス カウンタをジョブの種類によって分類します。カウンタ名は、PDH ヘルパ ファイルにおけるものと一致しています。IIS カウンタは読み取り専用です。カウンタとその説明に関するアルファベット順の一覧を表示するには、次の表でカウンタをクリックします。パフォーマンス モニタの詳細については、「パフォーマンス チューニングについて」を参照してください。
