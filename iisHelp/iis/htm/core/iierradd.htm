カスタム エラー メッセージを追加する

IIS スナップインの [カスタム エラー] プロパティ シートを使用して、カスタム エラー メッセージをファイルまたは URL にマップできます。

ファイルへのマッピングでエラー メッセージをカスタマイズするには

  1. カスタム エラー メッセージを収めたファイルを作成し、ディレクトリに置きます。
  2. IIS スナップインで、HTTP エラー メッセージをカスタマイズする Web サイト、仮想ディレクトリ、ディレクトリ、またはファイルを選択し、[プロパティ] をクリックします。
  3. [カスタム エラー] プロパティ シートをクリックします。
  4. 変更する HTTP エラーをクリックします。
  5. [プロパティの編集] をクリックします。
  6. [メッセージの種類] ボックスの一覧の [ファイル] をクリックします。
  7. カスタマイズしたエラー メッセージを指すパス名およびファイル名を入力するか、または [参照] をクリックしてコンピュータのハード ディスクにあるファイルを指定します。
  8. [OK] をクリックします。

URL へのマッピングでエラー メッセージをカスタマイズするには

  1. カスタム エラー メッセージを収めたファイルを作成し、仮想ディレクトリに置きます。
  2. IIS スナップインで、HTTP エラー メッセージをカスタマイズする Web サイト、仮想ディレクトリ、ディレクトリ、またはファイルを選択し、[プロパティ] をクリックします。
  3. [カスタム エラー] プロパティ シートをクリックします。
  4. 変更する HTTP エラーをクリックします。
  5. [プロパティの編集] をクリックします。
  6. [メッセージの種類] ボックスの一覧の [URL] をクリックします。
  7. カスタマイズしたエラー メッセージを指す URL を入力します。URL には、仮想ディレクトリ名から始まる URL へのパスを入力します。
  8. [OK] をクリックします。

注

  • カスタム エラーを処理するように ASP ファイルを定義した場合、スクリプトはコンテキストを渡す URL として呼び出されます。たとえば、Handle405.asp という名前の .asp ファイルが HTTP 405 エラーを処理するように設定されている場合、ユーザーが Example.htm にアクセスを試みて 405 エラーが発生すると、Web ブラウザでユーザーが「http://example.microsoft.com/handle405.asp?405;http://example.microsoft.com/example.htm」と入力した場合と同じように Handle405.asp ファイルが実行されます。
  • 静的なカスタム エラー ファイル (HTML ファイル) を使用する場合は、常に [ファイル] オプションを選択する必要があります。ISAPI または ASP を使って、エラーを処理するアプリケーションを開発する場合は、[URL] オプションを選択します。エラー状態は、URL のパラメータとしてアプリケーションに渡されるので、HTTP ヘッダーの状態は、アプリケーション側で設定する必要があることに注意してください。アプリケーション側で設定していないと、HTTP 応答の状態は "HTTP 1.1 200 OK" になります。
  • Windows で Internet Explorer 5.0 を使用している場合、Internet Explorer では、次のカスタム エラーを Internet Explorer 自身が持っている HTTP エラーに置き換える可能性があります。カスタム エラーのファイル サイズが次に示すサイズより小さい場合、カスタム エラーは使用されません。

    403、405、410 には、256 バイトより大きいファイル サイズが必要です。
    400、404、406、408、409、500、500.12、500.13、500.15、501、505 には、512 バイトより大きいファイル サイズが必要です。
