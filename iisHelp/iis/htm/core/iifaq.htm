よく寄せられる質問

  1. ドキュメントや Web ページを公開するには、どうすればよいですか。
  2. 仮想ディレクトリを Web サイトや FTP サイトに作成するには、どうすればよいですか。
  3. Web サイトを作成するには、どうすればよいですか。
  4. 複数の Web サイトを作成するには、どうすればよいですか。
  5. インターネット サービスを停止したり再開したりするには、どうすればよいですか。
  6. サイトの利用者の中に、それぞれのブラウザに合わせて設計した Web ページを見ることができない人がいるのはなぜですか。
  7. 障害のあるユーザーのためのユーザー補助機能に関する情報はどこに記載されていますか。
  8. IIS のテクニカル サポートを受けるには、どうすればよいですか。
  9. Web サイトの名前を変更できますか。また、旧サイト名宛ての要求を新しいサイトにリダイレクトできますか。
  10. ドライブが NTFS を使用するように構成されているかどうかを確認するには、どうすればよいですか。
  11. ファイル アロケーション テーブル (FAT) のファイル システム形式のドライブまたはパーティションを NTFS 形式に変換するには、どうすればよいですか。
  12. ユーザーごとに異なるアクセス レベルを設定して FTP サイトをセットアップできますか。
  13. サーバー証明書が特定の Web サイトにインストールされていることを確認するには、どうすればよいですか。
  14. 複数のサーバー証明書を 1 つの Web サイトにインストールできますか。
  15. 1 つのサーバー証明書を複数のサイトにインストールできますか。
  16. サーバー証明書を FTP サイトにインストールできますか。
  17. サーバー証明書のバックアップ コピーを作成する必要はありますか。
  18. FTP サービスに接続したユーザーの人数を確認するには、どうすればよいですか。
  1. ドキュメントや Web ページを公開するには、どうすればよいですか。

    2. ドキュメントを公開するには、ファイルを \inetpub\wwwroot ディレクトリに置きます。次に、ブラウザのアドレス バーに 「http://servername/filename」と入力して、公開したファイルを表示します。詳細については、「IIS によるサイトのクイック セットアップ」を参照してください。

  2. 仮想ディレクトリを Web サイトや FTP サイトに作成するには、どうすればよいですか。

    3. IIS スナップインを使用して、Web サイトに仮想ディレクトリを作成できます。「仮想ディレクトリを作成する」に、手順の説明があります。

  3. Web サイトを作成するには、どうすればよいですか。

    4. Microsoft® Windows XP に IIS をインストールすると、既定の Web サイトがセットアップされます。この既定の場所で、すぐにコンテンツを発行できます。

  4. 複数の Web サイトを作成するには、どうすればよいですか。

    5. IIS が組み込まれた Windows XP Professional は、単一のコンピュータ上で Web サイトと FTP サイトを 1 つずつホストできます。単一のコンピュータで複数の Web サイトや FTP サイトをホストする場合は、Windows Server 製品にアップグレードすることをお勧めします。

  5. インターネット サービスを停止したり再開したりするには、どうすればよいですか。

    6. インターネット サービスには、単独のプロセスで実行されるもの、またいくつかのプロセスによって構成されるものがあります。このため、インターネット サービスを最も効果的に停止および再開するには、ほかの Windows サービスとは異なる方法が必要です。IISReset.exe の使い方については、「IIS を再起動する」を参照してください。

  6. サイトの利用者の中に、それぞれのブラウザに合わせて設計した Web ページを見ることができない人がいるのはなぜですか。

    7. Browser Capabilities コンポーネントを使ってクライアントの表示機能を判別している場合は、ブラウザの新しいバージョンがリリースされるたびに、サーバーの Browscap.ini ファイルを定期的に更新する必要があります。Browscap.ini の更新バージョンは、Windows XP CD から取得するか、Microsoft の Web サイトからダウンロードできます。

  7. 障害のあるユーザーのためのユーザー補助機能に関する情報はどこに記載されていますか。

    8. Windows のユーザー補助機能に関する一般的な説明については、Windows のマニュアルを参照してください。特に IIS に関連するユーザー補助機能については、「IIS へのアクセス方法」を参照してください。

  8. IIS のテクニカル サポートを受けるには、どうすればよいですか。

    9. テクニカル サポートについては、このソフトウェアが相手先ブランド供給 (OEM) パッケージに含まれていた場合、またはパーソナル コンピュータの購入時に別途含まれていた場合は、コンピュータまたはサーバーの製造元にお問い合わせください。製造元の連絡先については、コンピュータやサーバーに同梱のマニュアルを参照してください。

    この製品を個別に販売店で購入した場合、オンライン マニュアルを参照しても解決できない技術的な質問については、Microsoft 製品サポート サービスの Web ページを参照してください。

  9. Web サイトの名前を変更できますか。また、旧サイト名宛ての要求を新しいサイトにリダイレクトできますか。

    10. ネットワーク上で正常に要求をルートできる場合は、Web サイトが新旧両方の名前宛ての要求に応答するように Web サイトを構成することができます。この構成により、古い名前でもサイトが使用でき、サイトの利用者には新しい名前が通知されます。名前を割り当てる方法については、「Web サイトに名前を付ける」を参照してください。

  10. ドライブが NTFS を使用するように構成されているかどうかを確認するには、どうすればよいですか。

    11. Windows エクスプローラで、ドライブを選択し、そのプロパティ シートを開きます。[全般] タブに、ファイル システムの種類が一覧表示されます。

  11. ファイル アロケーション テーブル (FAT) のファイル システム形式のドライブまたはパーティションを NTFS 形式に変換するには、どうすればよいですか。

    12. コマンド ラインに、「convert drive letter: /FS:NTFS」と入力します。NTFS ドライブを FAT に戻すことはできません。詳細については、Windows のマニュアルを参照してください。

  12. ユーザーごとに異なるアクセス レベルを設定して FTP サイトをセットアップできますか。

    13. NTFS ファイル システムでは、単一の仮想ディレクトリを使用し、個々の Windows ユーザー アカウントに、異なる種類のアクセスを許可または拒否するよう、NTFS のアクセス権を設定することができます。

    IP アドレスとドメイン名に基づいてアクセスを許可または拒否する場合は、Windows Server 製品のいずれかにアップグレードすることをお勧めします。

  13. サーバー証明書が特定の Web サイトにインストールされていることを確認するには、どうすればよいですか。

    14. サイトを選択し、そのプロパティ シートを開きます。[ディレクトリ セキュリティ] プロパティ シートの [セキュリティ保護された通信] に [編集] があります。このボタンが有効になっている場合は、Web サイトに証明書がインストールされています。ボタンが淡色表示になっている場合、Secured Sockets Layer (SSL) 機能を使用するには、サーバー証明書をサイトにインストールする必要があります。

  14. 複数のサーバー証明書を 1 つの Web サイトにインストールできますか。

    15. いいえ、できません。1 つの Web サイトには、サーバー証明書を 1 つだけインストールできます。

  15. 1 つのサーバー証明書を複数のサイトにインストールできますか。

    16. はい、必要です。サーバー証明書は、必要に応じて任意の数の Web サイトにインストールできます。

  16. サーバー証明書を FTP サイトにインストールできますか。

    17. いいえ、できません。FTP サイトは SSL 機能をサポートしていません。

  17. サーバー証明書のバックアップ コピーを作成する必要はありますか。

    18. はい、必要です。サーバー証明書は重要なものであり、サーバーの SSL 機能を使うための鍵となるものです。サーバー証明書のバックアップ コピーを作成するには、証明書全体をフロッピー ディスクにコピーして、安全な場所に保管してください。

  18. FTP サービスに接続したユーザーの人数を確認するには、どうすればよいですか。

    19. IIS スナップインで、FTP サイトを選択し、そのプロパティ シートを開いて、[現在のセッション] をクリックします。詳細については、「パフォーマンス チューニング」を参照してください。
