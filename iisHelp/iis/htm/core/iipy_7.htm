[セキュリティで保護された通信]

URL が https:// ではじまるような、保護された通信をサポートする Web ブラウザを使用する利用者のために、Secure Sockets Layer (SSL) による暗号化通信を構成します。

Web サーバーの SSL セキュリティ機能を使用する前に、"証明機関" と呼ばれる信頼できる第三者組織から有効なサーバー証明書を取得する必要があります。インターネット サービス マネージャ (HTML) では、この証明書を取得できないので、IIS スナップインを使用する必要があります。

[セキュリティで保護されたチャンネル (SSL) を要求] Web サイト、ディレクトリ、またはファイルへ接続する Web ブラウザに対して、暗号化された通信リンクを要求するには、このオプションを選択します。

[128 ビット暗号化を要求する] Web サイト、ディレクトリ、またはファイルへ接続する Web ブラウザに対して、128 ビットの暗号化された通信リンクを要求するには、このオプションを選択します。

重要   輸出規制により、128 ビット キーを使った暗号化機能を利用できるのは、米国およびカナダに限られています。

クライアント証明書

特定のファイルまたはディレクトリに接続するため、クライアント証明を承認、要求、または拒否するように Web サーバーを構成することができます。クライアント証明書は、"証明機関" と呼ばれる信頼できる第三者組織が発行するデジタル ID です。

[クライアント証明書を受諾しない] クライアント証明書の提示を要求せずにユーザーのアクセスを許可するには、このチェック ボックスをオンにします。

[証明書を受諾する] クライアント証明書を使ったアクセスを許可する一方で、証明書の要求を行わない場合は、このオプションを選択します。ユーザーのクライアント証明書は、マッピングに使用できます。クライアント証明書を持たないユーザーは、ほかの認証方式を使用します。

[クライアント証明書を要求] 有効なクライアント証明書を持つユーザーのみが接続できるようにするには、このチェック ボックスをオンにします。有効なクライアント証明書を持たないユーザーは、アクセスを拒否されます。このオプションを使用するには、[セキュリティで保護されたチャンネル (SSL) を要求] チェック ボックスをオンにする必要があります。

[クライアント証明書マッピングを有効にする]

有効なクライアント証明書を使ってログオンするユーザーを認証するには、このチェック ボックスをオンにします。マッピングでは、クライアント証明書に含まれている情報を、Windows ユーザー アカウントの情報と照合します。
