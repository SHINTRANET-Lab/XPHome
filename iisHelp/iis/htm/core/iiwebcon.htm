Web サイト管理

サイトがイントラネット上にある場合でも、インターネット上にある場合でも、コンテンツを提供するときの原理は同じです。サーバー上のディレクトリに Web ファイルを置くと、ユーザーが HTTP 接続をして、そのファイルを Web ブラウザで表示できるようになります。ただし、単にサーバーにファイルを格納するだけではなく、どのようにサイトを配置するかを検討する必要があります。さらに重要なことは、サイトの展開の仕方を検討することです。現在人気のある Web サイトで、静的なページで構成されたサイトはほとんどなく、成功している Web の管理者の多くは、常に変化のある Web コンテンツを提供し続けています。

ここでは、要求のリダイレクト方法から Web ページを動的に変更する方法まで、Web サイトのインフラ構造を管理するための基本について説明します。

ここで説明する項目は、次のとおりです。

  • 「Web サイト管理について」では、Web サイトを配置し、管理するプロセスを紹介します。
  • 「オーサリング ツールの選択」では、ニーズに合ったオーサリング ツールの選択方法の概念を説明します。
    • .
  • 「ASP を使用して Web サイト コンテンツを管理する」では、Microsoft Active Server Pages (ASP) を紹介し、これを使って Web コンテンツを管理する方法を説明します。
  • 「ホーム ディレクトリを変更する」では、サイトの中心となるディレクトリの指定について説明します。
  • 「仮想ディレクトリを作成する」では、サイトへのディレクトリの追加方法について説明します。
  • 「要求をディレクトリにリダイレクトする」では、ブラウザの要求をほかのディレクトリに転送する方法について説明します。
  • 「要求をプログラムにリダイレクトする」では、ブラウザの要求をアプリケーションに転送する方法について説明します。
  • 「コンテンツの有効期限を設定する」では、期間が重要な意味を持つ情報に対して有効期限を設定する方法について説明します。
  • 「コンテンツの規制を使用する」では、ユーザーが有害な内容を避けられるようにサイトの内容を規制する方法について説明します。
  • 「Web ページにフッターを追加する」では、サイトの各 HTML ページに自動的にフッターを挿入する方法について説明します。
  • 「サーバー側インクルード」では、ページが要求されてからブラウザに送られるまでに処理されるサーバー コマンドを、Web ページに挿入する方法について説明します。
