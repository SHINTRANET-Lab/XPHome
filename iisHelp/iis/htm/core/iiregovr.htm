レジストリについて

インターネット サービスを構成するには、IIS スナップインを使用します。IIS スナップインの構成にはないレジストリ内の構成パラメータには、サービス固有のエントリと、IIS およびすべてのサービスに適用されるグローバル エントリがあります。

インターネット サーバーの設定を変更する場合は、できる限り IIS スナップインを使用してください。レジストリの変更を有効にするには、対象となるサービスを再開する必要があります。グローバル エントリの場合は、すべてのサービスを再開する必要があります。

重要   レジストリ エディタの使い方を誤ると、Web サイトや FTP サイトに障害が起きるなどの問題が生じます。誤った設定を行うと、Web サイトや FTP サイトの構成が壊れることがあります。レジストリ エントリの編集は、ユーザー インターフェイスで調整できない設定を変更する場合のみ行ってください。レジストリを直接編集するときは、常に注意が必要です。

旧バージョンの IIS に存在した多数のレジストリ キーは、"メタベース" と呼ばれる、高速アクセスが可能な階層構造のデータベースに移行されています。移行されていないレジストリ キーは、インターネット サービスの開始と、これまでのバージョンの IIS との互換性を保つために使われます。新しい情報は、すべてメタベースに書き込まれます。詳細については、「メタベース」を参照してください。

レジストリ プロパティ テーブルのキー

レジストリのプロパティを表形式で表します。構成情報の編成は、次のとおりです。

識別子 : レジストリのプロパティを表す名前 (DWORD)。
データ型 : データのさまざまな型を示す DWORD
範囲 : プロパティに設定できる値の範囲
既定値 : ユーザーによる変更がない場合にプロパティに設定される値
プロパティの説明と、プロパティの変更による効果
