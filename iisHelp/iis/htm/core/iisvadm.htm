サーバーの管理

サーバーの管理に関する情報を示します。ここでは、既定の設定を変更する方法について説明します。また、サイトのトラフィックのログを収集し、サーバーのパフォーマンスを監視する方法についても説明します。

IIS の既定の設定を使用することも、また Web を公開するときのニーズに合うようにこれらの設定をカスタマイズすることもできます。既定の設定を調整して、付加価値を提供し、パフォーマンスを向上させ、セキュリティを強化することも可能です。

ここで説明する項目は、次のとおりです。

  • 「Web サイトと FTP サイト」では、Web サイトと FTP サイトの操作方法、リモート管理、およびサイトの名前付け方法などについて説明します。
  • 「WebDAV の発行」では、WebDAV を使用した Web サイトへのコンテンツの発行、HTTP 1.1 の実装方法について説明します。.
  • 「名前解決」では、"わかりやすい" 名前が IP アドレスに正しくマップされるように、WINS や DNS の名前解決システムをセットアップする方法について説明します。
  • 「アプリケーションを構成する」では、最大の信頼性とパフォーマンスを確保するために、アプリケーションの範囲を制御する方法について説明します。
  • 「セキュリティ」では、サーバー、コンテンツ、およびアプリケーションを不正な侵入から保護する方法について説明します。
  • 「カスタム エラー メッセージ」では、ユーザーが使いやすくカスタマイズできる HTTP 1.1 エラー メッセージについて説明します。
  • 「サイトの利用状況のログを収集する」では、サイトのトラフィックを追跡し、ログ ファイルの意味について説明します。
  • 「パフォーマンス チューニング」では、サーバーのパフォーマンスを監視し、向上させる方法について説明します。
  • 「管理スクリプト」では、コマンドライン スクリプトを使ってサーバーを管理する方法について説明します。

関連するその他の項目

  • スクリプトについては、「Active Server Pages」を参照してください。
