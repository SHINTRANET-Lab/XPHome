コンテンツの有効期限を設定する

期間を限定すべき情報が Web サイトに含まれている場合に、期間が過ぎた情報を発信しないように設定することができます。ある時点で、Web サイトのコンテンツが、自動的に期限切れになるように構成するには、[HTTP ヘッダー] プロパティ シートを使用します。コンテンツの有効期限が設定されている場合、Web ブラウザは現在の日付と有効期限の日付を比較し、キャッシュされたページを表示するか、更新されたページをサーバーに要求するかどうかを判断します。

Web サイトのコンテンツに有効期限を設定するには
  1. IIS スナップインで、コンテンツの有効期限を設定する Web サイト、仮想ディレクトリ、ディレクトリ、またはファイルを選択します。
  2. Web サイト、仮想ディレクトリ、ディレクトリ、またはファイルを右クリックし、[プロパティ] をクリックします。
  3. [HTTP ヘッダー] プロパティ シートをクリックします。
  4. [コンテンツに有効期限を設定する] チェック ボックスをオンにします。
  5. [直ちに有効期限を切る]、[有効期間]、または [有効期限] をクリックし、対応するボックスに適切な有効期限の情報を入力します。

注   通常、期間限定の情報は、個々のファイル、ディレクトリ、または Web サイトに限られています。ただし、WWW のマスタ プロパティを構成して、コンピュータ上のすべての Web サイトにコンテンツの有効期限を設定することもできます。詳細については、「継承された既定の設定を変更する」を参照してください。
