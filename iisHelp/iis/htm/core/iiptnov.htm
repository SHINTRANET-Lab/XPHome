パフォーマンス チューニング

インターネット インフォメーション サービス 5.1 の同時クライアント接続は、10 に制限されています。同時クライアント接続が 10 であれば、パフォーマンス チューニングは通常は必要ありません。このため、次の情報は数千の同時クライアント接続を処理できる製品である Windows 2000 Server および Windows 2000 DataCenter 以降を使用するユーザー向けです。IIS 5.1 のパフォーマンス チューニングに興味がある場合は、Active Server Pages (ASP) を使用したアプリケーション作成のパフォーマンス向上も考慮してください。詳細については、「Active Server Pages ガイド」を参照してください。

ここでは、サーバーとサイト両方のパフォーマンスを監視するための、Windows XP ツールの選択方法および使用方法について説明します。また、さまざまな設定を調整して、全体のパフォーマンスを向上させる方法についても説明します。IIS では、2 つの方法でパフォーマンスを設定できます。1 つは、IIS スナップインを使用する方法、もう 1 つはプログラムを使用して、メタベースの設定を直接変更する方法です。ここでは、IIS スナップインを使用してパフォーマンス設定を変更する方法のみを説明します。

ここで説明する項目は、次のとおりです。

  • 「パフォーマンス チューニングについて」では、パフォーマンスに関する概念を説明します。
  • 「ネットワーク容量」では、ネットワーク容量のプランニングの方法と、接続のパフォーマンスを最大限に引き出す方法について説明します。

関連するその他の項目

  • プログラムを使った管理については、「IIS をプログラム管理する」を参照してください。
  • サイトのコンテンツの選択がパフォーマンスに与える影響については、「アプリケーションを構成する」を参照してください。
  • Web サイトのトラフィックを追跡する方法については、「サイトの利用状況のログを収集する」を参照してください。
