[ディレクトリのセキュリティ] プロパティ シート

このプロパティ シートは、Web サーバーのセキュリティ機能を設定するために使用します。

[匿名アクセスおよび認証コントロール]

Web サーバーの認証および匿名アクセスの機能を構成するには、[編集] をクリックします。制限されたコンテンツへのアクセスを許可する前に、ユーザーの身元を確認するように、これらの機能を使って Web サーバーを構成します。

サーバーがユーザーを認証できるようにするには、まず有効な Windows のユーザー アカウントを作成し、そのアカウントに対して Windows ファイル システム (NTFS) のディレクトリおよびファイルのアクセス権を構成する必要があります。

[IP アドレスとドメイン名の制限]

(この機能は、Windows 2000 Server のインストール環境でのみ利用できます。)

特定のユーザー、コンピュータ、コンピュータ グループ、またはドメインに対して、Web サイト、ディレクトリ、またはファイルへのアクセスを許可または禁止するには、[編集] をクリックします。

[セキュリティで保護された通信]

Web サーバーの SSL (Secure Sockets Layer) 機能を利用して保護された通信を構成するには、[編集] をクリックします。Web サーバーの通信保護機能を使用するには、管理しているコンピュータに、有効なサーバー証明書があらかじめインストールされていなければなりません。

通信保護機能によって、次のことが可能になります。

  • ユーザーがディレクトリやファイルへアクセスする際、暗号化によって、保護された接続を使うように要求できます。
  • Web サーバーのクライアント証明書マッピングおよび認証機能を設定できます。
  • クライアント証明書マッピングを有効または無効にすることができます。
