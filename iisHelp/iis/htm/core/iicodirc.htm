コンテンツの規制を使用する

Web サーバーのコンテンツ規制機能を構成して、Web ページの HTTP ヘッダーにページのコンテンツを示すラベルを埋め込むことができます。Microsoft Internet Explorer Version 3.0 以降など一部の Web ブラウザでは、これらのコンテンツのラベルを検知し、好ましくない情報を含んでいると疑われる Web コンテンツを、ユーザーが識別できるようにします。Web サーバーの既定の Platform for Internet Content Selection (PICS) ベースの規制システムでは、Recreational Software Advisory Council (RSAC) が開発したシステムが使われています。 このシステムでは、暴力、性的な描写、および不快な言葉のレベルに応じてコンテンツを規制します。Web コンテンツの規制を設定するには、RSAC のアンケートに必要事項を記入し、その Web コンテンツに対して推奨されるコンテンツ規制を取得する必要があります。

コンテンツを規制するには
  1. IIS スナップインで、Web サイト、ディレクトリ、またはファイルを選択し、そのプロパティ シートを開きます。
  2. [HTTP ヘッダー] タブをクリックし、[コンテンツの規制] の [規制の編集] をクリックします。
  3. [規制サービス] プロパティ シートで、[RSAC 登録] の [規制のアンケート] をクリックします。RSAC の指示に従って操作します。
  4. [OK] をクリックします。

 

コンテンツの規制を設定するには
  1. IIS スナップインで、Web サイト、ディレクトリ、またはファイルを選択し、そのプロパティ シートを開きます。
  2. [HTTP ヘッダー] タブをクリックし、[コンテンツの規制] の [規制の編集] をクリックします。
  3. [規制] プロパティ シートで、[このリソースの規制を有効にする] チェック ボックスをオンにします。
  4. [分類] ボックスで、規制のカテゴリをクリックします。スライダを使用し、そのカテゴリについて不快感のレベルを設定します。各設定で、規制レベルの説明が表示されます。
  5. [このコンテンツを規制した人の電子メール名] ボックスに、内容を規制した人の電子メール名を入力します。[有効期限] ボックスのカレンダーから規制の有効期限を選びます。
  6. [OK] をクリックします。
