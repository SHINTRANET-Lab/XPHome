[ディレクトリ] プロパティ シート

このプロパティ シートは、Web サイトの物理ディレクトリのプロパティを変更するときに使用します。

物理ディレクトリは、次のいずれかからコンテンツを返します。

  • [構成中のサーバーにあるディレクトリ]  たとえば、C:\Catalog\Shoes などです。
  • [URL へのリダイレクト]  たとえば、http://tempserver などです。元の URL にあるリソースを要求するブラウザには、リソースを再要求できるように、[以下へリダイレクト] ボックスで指定されたリダイレクト先の URL が自動的に送信されます。要求を仮想ディレクトリにマップするには、/newcatalog などの仮想パスを使用します。

[ローカル パス]  ディレクトリの場所を表示します。

[アクセス権]

[読み取り]  ファイルやディレクトリ、およびそれらに関連するプロパティの読み取りとダウンロードを許可するには、このチェック ボックスをオンにします。

[書き込み]  ファイルおよびファイルに関連するプロパティを、サーバー上の有効なディレクトリへアップロードする、または書き込み可能なファイルの内容の変更を許可するには、このチェック ボックスをオンにします。[書き込み] が使用できるのは、HTTP 1.1 プロトコル規格の PUT 機能をサポートするブラウザを使用する場合のみです。

[スクリプト ソース アクセス]  [読み取り] または [書き込み] のアクセス権が設定されている場合に、ソース コードへアクセスできるようにするには、このチェック ボックスをオンにします。ソース コードには、ASP アプリケーション内のスクリプトが含まれます。

[ディレクトリの参照]  仮想ディレクトリ内のファイルとサブディレクトリの一覧をハイパーテキスト形式で表示できるようにするには、このチェック ボックスをオンにします。この一覧に仮想ディレクトリは表示されません。ユーザーは、あらかじめ仮想ディレクトリのエイリアスを知っておく必要があります。

重要   次の "両方" が当てはまる場合に、ユーザーがファイルまたはディレクトリにアクセスしようとすると、Web サーバーは "Access Forbidden" というエラー メッセージを Web ブラウザに表示します。

  • ディレクトリの参照が無効になっている。

  • filename.htm などのファイル名が指定されていない。

[ログ アクセス]  ディレクトリへのアクセスをログ ファイルに記録するには、このチェック ボックスをオンにします。アクセスが記録されるのは、Web サイトでログの収集が有効になっている場合のみです。

[このリソースを索引付けする]  Microsoft インデックス サービスが、このディレクトリを Web サイトのフルテキスト インデックスの形式で取り込めるようにするには、このチェック ボックスをオンにします。フルテキスト インデックスを使用すると、ユーザーは、公開されたドキュメントから単語や語句をすばやく検索できます。

[アプリケーションの設定]

IIS の Web ベースのアプリケーションは、アプリケーションが置かれているディレクトリ構造によって定義されます。このディレクトリをアプリケーションの開始点としてアプリケーションを作成するには、[作成] をクリックします。このホーム ディレクトリとアプリケーションを分離するには、[削除] をクリックします。

[実行アクセス権]

実行アクセス権によって、サイトや仮想ディレクトリのリソースに許可されるプログラムの実行レベルが決まります。

  • [なし]  HTML ファイルやイメージ ファイルなどの静的ファイルだけにアクセスできます。
  • [スクリプトのみ]  ASP スクリプトなどのスクリプトだけを実行できます。
  • [スクリプトおよび実行可能ファイル]  どのファイル タイプでもアクセスまたは実行することができます。

アプリケーション保護

アプリケーションを、Web サービスと同じプロセスで実行するか ([低 (IIS プロセス)])、ほかのアプリケーションも実行される分離されたプール プロセスで実行するか ([中 (プール)])、またはほかのプロセスから切り離された分離プロセスで実行するか ([高 (分離プロセス)]) を選択します。

[クライアントは以下に送信されます]

以下のプロパティは、[URL へのリダイレクト] を選択した場合に表示されます。

[上で入力した URL]  元の URL とまったく違うかたちで、指定された URL に仮想ディレクトリをリダイレクトします。このオプションを使用すると、仮想ディレクトリ全体を 1 つのファイルにリダイレクトすることができます。たとえば、/scripts という仮想ディレクトリに対する要求を、すべてホーム ディレクトリの Default.htm というファイルにリダイレクトするには、[以下へリダイレクト] ボックスに「/Default.htm」と入力し、このチェック ボックスをオンにします。

[この下のディレクトリ]  親ディレクトリを子ディレクトリにリダイレクトします。たとえば、/ で表されるホーム ディレクトリを、/newhome という名前のサブディレクトリにリダイレクトするには、[以下へリダイレクト] ボックスに「/newhome」と入力し、このチェック ボックスをオンにします。チェック ボックスをオフにすると、Web サーバーは引き続き親ディレクトリを親自体にマップします。

[このリソースへの恒久的なリダイレクト]  クライアントに "301 Permanent Redirect" というメッセージを送信します。リダイレクトが一時的と見なされた場合、クライアント ブラウザは、"302 Temporary Redirect" というメッセージを受信します。ブラウザによっては、ブックマークなどの URL が恒久的に変更された合図として、"301 Permanent Redirect" というメッセージを使用できる場合があります。
