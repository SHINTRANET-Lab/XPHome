[認証方法]

このダイアログ ボックスは、ユーザーを識別、つまり "認証" するように Web サーバーを構成するために使用します。これにより、権限のないユーザーがアクセスの制限されたコンテンツに Web (HTTP) 接続することを防ぎます。認証処理では、ユーザーが、特定の Web サイト、ディレクトリ、またはファイルにアクセスするための、適切な Windows ファイルシステムのアクセス権が付加されている、有効な Windows ユーザー アカウントを持っているかどうかを確認します。

[匿名接続]

ユーザーが匿名で接続できるようにするには、このチェック ボックスをオンにします。有効な Windows ユーザー アカウントである匿名アカウントまたはゲスト アカウントでユーザーがログオンできます。

[認証済みアクセス]

[基本認証]

ユーザーの識別に広く使用されている業界標準の基本認証方式を、Web サーバーで有効にするには、このチェック ボックスをオンにします。

重要   基本認証では、パスワードが暗号化されずにネットワーク上を伝送されます。そのため、悪意のある第三者がネットワーク監視ツールを使用して、ユーザー名とパスワードを意図的に傍受するおそれがあります。

[編集]

ドメイン名を明示しないユーザーに対し、ローカル ドメインではなく、既定のログオン ドメインと見なすように Web サーバーを構成するには、[編集] をクリックして設定します。

[ダイジェスト認証]

IIS 5.0 および 5.1 で新しく導入されたこの方式では、パスワードではなくハッシュ値をネットワーク上に送ります。この方式は、プロキシ サーバーやその他のファイアウォールで使用することができます。

[統合 Windows 認証]

統合 Windows 認証処理では、Web サーバーは、ユーザーの Internet Explorer Web ブラウザと暗号情報をやり取りします。

統合 Windows 認証をいったん有効にすると、Web サーバーは、次の場合のみこの認証処理を使用します。

  • 匿名アクセスが無効になっている。
  • Windows ファイル システムのアクセス権が設定されているために、制限されたコンテンツへの接続時に Windows ユーザー名とパスワードの入力が必要となり、その結果匿名アクセスが拒否されている。
