証明書

証明書は、サーバーの情報を要求するクライアント、およびサーバーの、デジタル識別情報の 1 つの形式です。証明書は、パスポートなどの公式な ID と同じように、所有者を識別する機能を果たします。証明書は、機密情報を送信できる安全な接続を確立する Secure Sockets Layer (SSL) 機能の一部です。

ここでは、次の項目について説明します。

  • 「証明書について」では、証明書の概要について説明します。
  • 「サーバーに SSL をセットアップする」では、Secure Sockets Layer (SSL) をサーバーにセットアップし、管理する方法について説明します。
  • 「新しいセキュリティ機能ウィザードを使用する」では、サーバー証明書ウィザード、証明書信頼リスト ウィザード、およびアクセス許可ウィザードの機能の概要について説明します。
  • 「サーバー証明書を取得する」では、SSL の使用に必要なサーバー証明書の取得方法について説明します。
  • 「証明書信頼リストを使用する」では、サーバーの信頼できる証明書発行機関の一覧に、新しい証明機関を追加する方法について説明します。
  • 「クライアント証明書を取得する」では、クライアント証明書と呼ばれるユーザーのデジタル識別情報を取得する方法について説明します。
  • 「クライアント証明書を有効にする」では、クライアント証明書の SSL 認証を有効にする手順について説明します。
  • 「ユーザー アカウントにクライアント証明書をマップする」では、クライアント証明書を使用してユーザーを直接 Windows ユーザー アカウントにマップする方法について説明します。
