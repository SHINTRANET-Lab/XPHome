変更箇所

ここでは、IIS 4.0、IIS 5.0、および IIS 5.1 の既定の動作および設定に関する、重要な変更箇所について説明します。

IIS 5.1 の新機能については、「機能」を参照してください。

管理

  • IIS 4.0 では、アプリケーションをインターネット サービスと同じプロセス、または別のプロセスで実行していました。IIS 5.0 および 5.1 では、アプリケーションをプール プロセスにグループ化して、パフォーマンスと拡張性を向上させることができるようになりました。詳細については、「アプリケーションについて」の「アプリケーション保護」を参照してください。
  • [アプリケーションのマッピング] プロパティ シートには、特定のファイル タイプにマップされたアプリケーションによって処理される HTTP (Hypertext Transfer Protocol) 動詞の一覧が表示されます。この動詞の一覧が、IIS 4.0 からの変更箇所です。IIS 4.0 では、この一覧に、"排除される" 動詞、つまり処理されない動詞が表示されていました。この変更は、プロトコルに追加される新しい HTTP 動詞に対応するために行われました。アプリケーションのマッピングの詳細については、「アプリケーションのマッピングを設定する」を参照してください。
  • カスタム エラー ファイルの場所が変わりました。詳細については、「詳細なカスタム エラー メッセージを有効にする」を参照してください。
  • インターネット サービス マネージャは、[スタート] メニューからアクセスできなくなりました。IIS を管理するには、IIS スナップイン を開きます。

セキュリティ

  • IIS 4.0 で提供されていたセキュリティ機能の多くは、IIS 5.0 および 5.1 で簡略化されました。IIS 5.0 には、アクセス許可ウィザード、サーバー証明書ウィザード、および証明書信頼リスト ウィザードの 3 つの新しいセキュリティ機能ウィザードがあります。これらのウィザードを使用して、Web と NTFS のセキュリティ設定の同期化、サーバー証明書の取得とインストール、および証明書信頼リストの作成と変更を行うことができます。詳細については、「新しいセキュリティ機能ウィザードを使用する」を参照してください。
  • [このリソースに索引を付ける] チェック ボックスは、既定で有効になっています。
  • ASP ページやその他のスクリプトの "ソース コード" へアクセスできるようにする [スクリプト ソース アクセス] チェック ボックスが新しく追加され、既定で無効になっています。このチェック ボックスは、[読み取り] または [書き込み] チェック ボックスのいずれかがオンになっている場合にのみ利用できます。
  • FrontPage を使ってサーバーのサイトを管理できるようにするための [FrontPage Web] チェック ボックスは、IIS スナップインでは設定できなくなりました。 この機能は、既定で有効になっています。
