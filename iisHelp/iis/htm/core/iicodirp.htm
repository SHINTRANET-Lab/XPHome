要求をプログラムにリダイレクトする

特定のディレクトリにある複数のファイルへの要求を、すべて 1 つのプログラムにリダイレクトできます。 これは、リダイレクトの高度な使用方法です。通常は、リダイレクトの変数を使用して、元の URL で指定されているパラメータをプログラムに渡すこともできます。

要求をプログラムにリダイレクトするには
  1. IIS スナップインで、Web サイトまたはディレクトリを選択し、そのプロパティ シートを開きます。
  2. [ホーム ディレクトリ] タブ、[仮想ディレクトリ] タブ、または [ディレクトリ] タブをクリックします。
  3. [URL へのリダイレクト] をクリックします。
  4. [リダイレクト先] ボックスに、プログラムの URL と、パラメータをプログラムに渡すために必要なリダイレクトの変数を入力します。

    5. たとえば、Scripts ディレクトリのスクリプトに対するすべての要求を、要求された URL および URL と共に渡されたパラメータを記録するログ収集プログラムにリダイレクトするには、「/Scripts/Logger.exe?URL=$V+PARAMS=$P」と入力します。$V と $P は、リダイレクトの変数です。

  5. [上で入力した URL] チェック ボックスをオンにし、Web サーバーがリダイレクト先の URL に元のファイル名を追加しないようにします。

リダイレクトの変数の詳細については、「リダイレクト リファレンス」を参照してください。
