要求をディレクトリにリダイレクトする

あるディレクトリ内にある複数のファイルへの要求を、別のディレクトリ、別の Web サイト、または別のディレクトリの別のファイルにリダイレクトできます。ブラウザが元の URL のファイルを要求すると、Web サーバーは新しい URL を使用してページを要求するようブラウザに通知します。

ほかのディレクトリまたは Web サイトに要求をリダイレクトするには
  1. IIS スナップインで、Web サイトまたはディレクトリを選択し、そのプロパティ シートを開きます。
  2. [ホーム ディレクトリ] タブ、[仮想ディレクトリ] タブ、または [ディレクトリ] タブをクリックします。
  3. [URL へのリダイレクト] をクリックします。
  4. [リダイレクト先] ボックスに、リダイレクト先のディレクトリまたは Web サイトの URL を入力します。たとえば、/Catalog ディレクトリのファイルに対する要求をすべて /NewCatalog ディレクトリにリダイレクトするには、「/NewCatalog」と入力します。

 

すべての要求を単一のファイルにリダイレクトするには
  1. IIS スナップインで、Web サイトまたはディレクトリを選択し、そのプロパティ シートを開きます。
  2. [ホーム ディレクトリ] タブ、[仮想ディレクトリ] タブ、または [ディレクトリ] タブをクリックします。
  3. [URL へのリダイレクト] をクリックします。
  4. [リダイレクト先] ボックスに、リダイレクト先のファイルの URL を入力します。
  5. [上で入力した URL] チェック ボックスをオンにし、Web サーバーがリダイレクト先の URL に元のファイル名を追加しないようにします。

リダイレクト先の URL では、ワイルドカードとリダイレクトの変数を使用して、元の URL をリダイレクト先の URL に変換する方法を細かく制御できます。詳細については、「リダイレクト リファレンス」を参照してください。
