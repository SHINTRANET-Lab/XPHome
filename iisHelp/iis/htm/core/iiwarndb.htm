アプリケーションを構成する

ここでは、IIS スナップインを使って Web ベースのアプリケーションを構成する方法について説明します。

ここでは、次の項目について説明します。

  • 「アプリケーションについて」では、Web ベースのアプリケーションを定義し、サイトのディレクトリ構造がアプリケーションの定義とどのように関連しているかについて説明します。
  • 「アプリケーションを作成する」では、ディレクトリをアプリケーションの開始点、つまりルートとしてセットアップする方法について説明します。
  • 「アプリケーションのマッピングを設定する」では、ファイル名の拡張子を、そのファイルを処理するプログラムに関連付ける方法について説明します。
  • 「アプリケーションを分離する」では、アプリケーションを Web サーバーとは別のメモリ領域で実行する方法について説明します。
  • 「アプリケーションをキャッシュする」では、CGI スクリプトのタイムアウトを設定し、ISAPI の DLL をキャッシュし、ASP ページのキャッシュを変更する方法について説明します。
  • 「分離プロセスで動作するアプリケーションを停止する」では、分離プロセスで動作するアプリケーションを停止する方法について説明します。
  • 「プロセスの再利用」では、アプリケーションの再利用方法、構成可能イベント、プログラミング、およびセットアップに関する考慮事項について説明します。
  • 「ASP アプリケーションを構成する」では、ASP アプリケーションのプロパティを設定する方法について説明します。
  • 「ASP のデバッグを有効にする」では、ASP ページのチェックと修正を行うために Web サーバーでデバッグを有効にする方法について説明します。
  • 「CGI アプリケーションを構成する」では、Web サーバーに CGI アプリケーションをインストールし、構成する方法について説明します。
  • 「ISAPI フィルタをインストールする」では、ISAPI フィルタについて、およびISAPI フィルタを Web サーバーのイベントに関連付ける方法について説明します。

関連するその他の項目

  • ASP アプリケーションを記述する方法については、「Active Server Pages」を参照してください。
  • 証明書の使い方については、「証明書」を参照してください。
