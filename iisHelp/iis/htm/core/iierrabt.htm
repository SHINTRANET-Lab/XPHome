カスタム エラー メッセージについて

ここでは、既定の HTTP 1.1 エラーおよびカスタマイズした HTTP 1.1 エラーについて概要を説明します。

  • HTTP 1.1 エラー メッセージ
  • カスタム エラー メッセージ

HTTP 1.1 エラー メッセージ

HTTP 1.1 エラー メッセージは、一般的なメッセージを含んだ HTML ページの形式でクライアントのブラウザに返されます。ユーザーが Web サイトに接続しようとしたときに HTTP エラーが発生すると、接続の確立中に何が発生したかを簡単に説明するエラー メッセージが、クライアントのブラウザに送信されます。たとえば、既に接続の処理能力の限界に達している Web サイトにユーザーが接続を試みると、"Too many users" というメッセージを含む HTTP エラーがクライアントに返されます。

カスタム エラー メッセージ

すべての IIS カスタム エラー メッセージは、業界標準の HTTP コードを表示し、HTTP 1.1 エラー メッセージとの一貫性を保つことができます。IIS の [カスタム エラー] プロパティ シートを使って、一般的な HTTP エラー メッセージをカスタマイズできます。

次の HTTP エラー メッセージを IIS でカスタマイズできます。

注

  • IIS スナップインでは、カスタム エラー メッセージはリストとして表示され、リスト全体が単一のプロパティとして扱われます。たとえば、カスタム エラー メッセージのセットを Web サイト レベルで構成すると、そのサーバーに属するすべてのディレクトリに、カスタム エラー メッセージのリストがすべて継承されます。つまり、サーバーとディレクトリの 2 つのカスタム エラー メッセージ リストは統合されません。
  • 404.1 エラーは、複数の IP アドレスを持つコンピュータでのみ発生します。クライアント要求が指定の IP アドレス/ポートの組み合わせで受信され、その IP アドレスが特定のポートでリッスンするように構成されていない場合、IIS は 404.1 HTTP エラーを返します。たとえば、コンピュータが 2 つの IP アドレスを持ち、その中の 1 つの IP アドレスだけがポート 80 でリッスンするように構成され、ポート番号 80 を使用して、もう 1 つの IP アドレスに受信を要求した場合、すべての要求に IIS は 404.1 エラーを返します。このエラーは、サーバー上で複数の IP アドレスが使われているときのみクライアントに返されるため、サービス レベルでのみ設定します。
