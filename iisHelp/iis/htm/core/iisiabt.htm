サーバー側インクルードについて

Web コンテンツをユーザーに送る直前に、テキスト、グラフィックス、またはアプリケーション コンテンツを Web ページにインクルードするには、サーバー側インクルード (SSI) ディレクティブを使用します。SSI を使い、タイムスタンプ、著作権情報、顧客が記入して返すフォームなどをインクルードできます。ファイルのインクルードによって、テキストやグラフィックスを多くのファイルに繰り返し取り込むことが容易にできます。各ファイルにコンテンツを入力するのではなく、コンテンツをインクルード ファイルに保存します。インクルード ファイルは、Web サーバーに該当する Web ページにコンテンツを挿入するよう命令する簡単なステートメントを使って呼び出します。さらに、インクルード ファイルを使用すると、一箇所の内容を変更するだけで済みます。

SSI ディレクティブが入ったファイルには特殊な処理が必要なため、SSI ファイルには SSI 用のファイル名拡張子を付ける必要があります。既定の拡張子は、.stm、.shtm、および .shtml です。Web サーバーが認識する一覧に拡張子を追加できます。詳細については、「アプリケーションのマッピングを設定する」を参照してください。

Web サーバーでは、Web ページの処理中に SSI ディレクティブを処理します。Web サーバーは、SSI ディレクティブに達すると、インクルード ファイルの内容を直接 HTML ページに挿入します。インクルード ファイルに SSI ディレクティブが入っている場合、そのファイルもまた挿入されます。ファイルのインクルードに使用される基本的なディレクティブのほかに、SSI ディレクティブを使って、ファイル サイズなどのファイルに関する情報の挿入、またはアプリケーションやシェル コマンドの実行ができます。

詳細については、「サーバー側インクルード リファレンス」を参照してください。
