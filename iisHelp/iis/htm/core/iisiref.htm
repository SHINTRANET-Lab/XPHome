サーバー側インクルード リファレンス

静的 Web ページで特定のタスクを実行するためにサーバー側インクルード (SSI) ディレクティブを使用します。静的 Web ページは、大規模な ASP ページよりもわずかに速く実行します。SSI ディレクティブを使用すると、ファイルの内容を Web ページに含めることができ、ファイルに関する情報、サーバー変数、または CGI ファイルや ISAPI アプリケーションなどの実行可能ファイルからの出力を含めることができます。

実行可能ファイルを含めることを除き、これらタスクのすべては ASP スクリプトを使用して実行できます。Web ページ内でほかのタスクに ASP スクリプトを使用する必要がある場合は、これらのタスクに ASP ソリューションを使用する必要があります。#include ディレクティブを除き、SSI ディレクティブと ASP スクリプトを一緒に使用することはできません。

注  サーバー側の時刻および日付関数では、オペレーティング システム ロケールではなく C ライブラリ ロケールで設定される言語を使用します。C ライブラリ ロケールは通常は英語に設定されています。

このセクションの各部について説明します。

  • 「#config」は、SSI エラー メッセージに使用される文字列および #flastmod や #fsize によって返される日付とファイル サイズに使用される形式を指定します。
  • 「#echo」は、サーバー変数の値を HTML ページに挿入します。
  • 「#exec」は、アプリケーションまたはシェル コマンドを実行し、その結果を HTML ページに挿入します。
  • 「#flastmod」は、#config ディレクティブで書式設定されたファイルの変更日付を Web ページに挿入します。
  • 「#fsize」は、#config ディレクティブで書式設定されたファイルのサイズを Web ページに挿入します。
  • 「#include」は、ファイルを Web ページにインクルードします。このディレクティブは、ASP ファイルと STM ファイルで使用できる唯一のディレクティブです。
