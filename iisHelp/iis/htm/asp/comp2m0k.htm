Database Access コンポーネント

Database Access コンポーネント (DAC) は、データベースまたはその他の表形式のデータ構造に格納された情報へのアクセスを提供します。 DAC には、ActiveX® Data Objects (ADO)、オープン データベース コネクティビティ (ODBC)、および Object Linking and Embedding for Databases (OLEDB) が含まれます。 ASP では、これらのコンポーネントを使用して Microsoft® Access、SQL、Excel およびその他のテクノロジなどのさまざまなデータベース ソースからの情報を簡単に統合できます。複数のタイプのデータ格納領域へのアクセスを、Universal Data Access と呼びます。

Universal Data Access を使用するアプリケーションの開発については、Universal Data Access の Web サイトまたは Platform SDK の「Microsoft Data Access SDK」を参照してください。

ADO は、簡単に使用できるため、最も一般的に使用される Data Access コンポーネントです。ADO には、データへのアクセスを支援するためのメソッドとプロパティを備えた 7 つのオブジェクトのライブラリがあります。各オブジェクトは、Server.CreateObject を使用して作成できます。

<% 
   ' Create two of the seven ADO objects.
   Set conn = Server.CreateObject("ADODB.Connection")
   Set rs = Server.CreateObject("ADODB.RecordSet")
%>

注

ADO では、タイプ ライブラリ内に定義済みの多数の定数を使用するので、ユーザーが特定の値を記憶する必要がありません。ADO Referenceに指定された定数を使用するには、ADO 定数に関する情報が入ったファイルをインクルードするか、または ADO タイプ ライブラリへの参照を Global.asa ファイルにインクルードする必要があります。次の例では、ADO タイプ ライブラリを使用して定数にアクセスします。

たとえば、この行を ASP ファイルで使用するには、3 は Open Static および Lock Optimistic を示すことを記憶する必要があります。

<% rs.Open "Customers", Conn, 3, 3 %>

ADO 用のタイプ ライブラリをインクルードする場合は、このコードを使用できます。

<% rs.Open "Customers", Conn, adOpenStatic, adLockOptimistic %>

アプリケーションに次のタイプ ライブラリをインクルードするには、この宣言を Global.asa ファイルに配置します。

<!-- METADATA TYPE="typelib" FILE="c:\program files\common files\system\ado\msado20.dll"-->

タイプ ライブラリと定数の詳細については、「変数と定数を使用する」を参照してください。データ アクセスの詳細については、「データ ソースにアクセスする」および「ASPでデータにアクセスする」を参照してください。
