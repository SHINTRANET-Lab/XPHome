コンポーネント サービス

コンポーネント サービスには、コンポーネントや Web アプリケーションの開発を容易にするために、次のようなサービスが提供されています。

キュー コンポーネント

キュー コンポーネントにより、クライアントとサーバーが接続されている場合に、即座に実行可能なコンポーネントを作成することができます。これにより非同期にコンポーネントを呼び出し、実行することが容易になります。クライアントとサーバーが接続されていない場合、その接続が確立するまでコンポーネントの実行は保留になります。キュー コンポーネントには、コンポーネント開発に使用されるメソッドの呼び出しに類似したメソッドの呼び出しが使用されるため、マーシャリングに関する深い知識を習得する苦労を軽減します。詳細については、Platform SDK の「Component Services」セクションの「Queued Components」を参照してください。

コンポーネント サービス イベント

コンポーネント サービス イベントによって、パブリッシャとサブスクライバはデータ ソースに緩やかに接続されるため、それぞれが個別にデータ ソースを開発、配備、および実行できるようになります。パブリッシャは、サブスクライバの位置や数を把握する必要はありません。また、サブスクライバは仲介ブローカーを介して、パブリッシャを見つけたり、パブリッシャへのサブスクリプションを管理したりすることができます。イベント システムは、パブリッシャとサブスクライバの両方の身元を永続的にすることで、Web アプリケーションの開発を容易にします。パブリッシャとサブスクライバは、互いを知らなくても、その身元を操作することができます。詳細については、Platform SDK の「Component Services」セクションの「COM+ Events」を参照してください。
