メタベース構造

IIS の構成設定のほとんどは、"メタベース" と呼ばれる、Windows レジストリに似た記憶機構に格納されています。メタベースの動作、メタベースにおける情報の格納方法、およびその情報にアクセスする方法についての知識が必要です。

メタベースは、IIS インストール環境の構造を反映した階層構造になっています。メタベース構造の各ノードは "キー" と呼ばれ、各キーには "メタベース プロパティ" と呼ばれる 1 つまたは複数の IIS 構成値が含まれています。IIS のメタベース キーは IIS の各要素に対応し、各キーには、関連する要素の構成に影響を与えるプロパティが含まれています。

IIS の以前のバージョンではシステム レジストリに格納されていた IIS 構成キーおよび値のほとんどが、プロパティとしてメタベースに格納されるようになりました。また、新しい値やキーが追加されて、IIS をより細かく、柔軟に制御できるようになりました。メタベースは、同じプロパティの設定をノードごとに変えられるように設計されています。たとえば、MaxBandWidth プロパティの設定を、サーバーごとに変えることができます。

次の図は、キー タイプ別に整理した IIS の一般的な構造です。IIS インストール環境のメタベース構造を構成する要素の数は、各インストール環境に依存します。IIS の要素に関連付けられたメタベース キーは、メタベースにおけるそのキーのパスで参照できます。「キー名とパス」を参照してください。

最上位のメタベース キーは、" IISComputer" という名前で、MaxBandWidth などのプロパティを含んでいます。これらのプロパティは、コンピュータにおける IIS の実行全般に影響します。Computer キーの 2 つのサブキーは、それぞれ FTP サービスと Web サービスに関連付けられていて、そのコンピュータにホストされているすべての FTP サーバーと Web サーバーに影響を与えるプロパティを含んでいます。

FTP サービス キーのサブキーは、個々の FTP サーバーと関連付けられ、それぞれの FTP サーバー固有のプロパティを含んでいます。各 FTP サーバー キーのサブキーの 1 つは、そのサーバーのルート仮想ディレクトリに関連付けられています。このルート仮想ディレクトリ キーには、FTP サーバーに仮想ディレクトリが追加されるたびにサブキーが追加されます。これらのサブキーはそれぞれ、関連する仮想ディレクトリの操作や構成に影響を与えるプロパティを含んでいます。FTP サービス キーに直接従属する Info キーにも、FTP サービスに関連するいくつかのプロパティが含まれます。

Web サービス キーのサブキーは、個々の Web サーバー、それらのルート仮想ディレクトリ、従属する仮想ディレクトリ、ディスク ディレクトリ、およびファイルに関連付けられています。Web サービス キーのサブキーは、Web サービスの操作で使用されるフィルタの構成に影響します。Web サービス キーに直接従属する Info キーにも、Web サービスに関連するいくつかのプロパティが含まれています。

最上位のキーに直接従属するその他のキーには、ログ収集と MIME マッピングに影響するプロパティが含まれています。IISADMIN キーは、DCOM の拡張機能を IIS に記録するために使用されます。

キーには、IIS Admin オブジェクトが使用するオブジェクトへの参照が含まれている場合もあります。たとえば CertMapper キーは、メタベース プロパティを含んでいませんが、IIsCertMapper オブジェクトおよび証明書をマップするそのオブジェクトのメソッドへのアクセスを提供します。
