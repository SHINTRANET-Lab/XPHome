ADO のクライアント ネットワーク ライブラリを選択する

ADO は、ネットワーク ライブラリを通してデータベースと通信します。選択するネットワーク ライブラリは、データ プロバイダとシステム構成によって決まり、データベースのアクセス速度に大きく影響します。たとえば、Microsoft SQL Server データベースからデータにアクセスする場合、一般に TCP/IP ネットワーク ライブラリの使用によってアクセス速度はより速くなります。ただし、SQL Server が IIS と同じコンピュータ上で実行されている場合は、名前付きパイプ ネットワーク ライブラリを使用することにより、パフォーマンスが向上する場合があります。

DSN のネットワーク ライブラリは、コントロール パネルのデータソース (ODBC) オプションを使用して設定できます。1 つの接続についてライブラリを設定すると、そのライブラリがその後のすべての接続、およびアプリケーションで使用される可能性のある DSN なしの接続のための既定のライブラリになります。

注   ネットワーク ライブラリを名前付きパイプから TCP/IP に変える場合は、名前付きパイプをオプションの 1 つとして削除しないように注意してください。これは、SQL Server Enterprise Manager が名前付きパイプ ライブラリに依存しているためです。
