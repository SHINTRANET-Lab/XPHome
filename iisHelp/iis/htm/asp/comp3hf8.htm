Permission Checker コンポーネント

Permission Checker コンポーネントは、PermissionChecker オブジェクトを作成します。このオブジェクトは、Microsoft® インターネット インフォメーション サービス (IIS) が提供するパスワード認証プロトコルを使用して、Web ユーザーにファイルを読み取る権限があるかどうかを判別します。

Permission Checker コンポーネントを使用して、ユーザーの種類に応じて ASP ベースのページをカスタマイズできます。たとえば、Web ページにハイパーリンクが含まれている場合、Permission Checker コンポーネントを使用すると、ユーザーがリンク先の Web ページへのアクセス権を持っているかどうかを調べることができます。ユーザーが適切なアクセス権を持っていない場合は、アクセスできないページへのハイパーリンクを省略したり、変更したりすることができます。

構文

Set oVar = Server.CreateObject( "MSWC.PermissionChecker" )

 

パラメータ
oVar
Server.CreateObject を呼び出して作成する PermissionChecker オブジェクトの名前を指定します。
メソッド
HasAccess ユーザーが、指定されたファイルへのアクセス権を持っているかどうかを調べます。 




注  



IIS では、次の 4 種類のパスワード認証をサポートしており、これらを任意に組み合わせて使用できます。


    
	
  • 匿名認証
    • 

	
  • 基本認証
    • 

	
  • 統合 Windows 認証
    • 




匿名認証が有効になっている場合、すべてのユーザーはまず IIS の匿名ユーザー アカウントでログオンします。この場合、すべての匿名ユーザーが同じアカウントを共有するため、匿名アクセスが許可されているときは Permissions Checker コンポーネントで個々のユーザーを認証することはできません。



イントラネット専用の Web サイトのように、すべてのユーザーが個別にアカウントを持っているアプリケーションでは、匿名認証を無効にし、Permissions Checker コンポーネントが個々のユーザーを認証できるようにすることをお勧めします。



インターネットとイントラネットを組み合わせた Web サイトのように、一部のページは匿名ユーザーが利用できるようにし、ほかのページは保護する必要があるアプリケーションでは、匿名認証を有効にしたうえで、統合 Windows 認証または基本認証のパスワード認証方式の少なくとも 1 つを有効にする必要があります。このように設定した後、特定のページで匿名アクセスを拒否すると、サーバーは統合 Windows 認証または基本認証のいずれかを使用してユーザーの認証を試みます。



特定のページへの匿名アクセスを拒否するには、次のいずれかの方法を使用できます。


    
	
  • ASP ベースのファイルのアクセス制御リストを設定し、匿名ユーザー アカウントを除外します。
    • 

	
  • ASP スクリプト内で、匿名ユーザー アカウントかどうか (LOGON_USER サーバー変数が空かどうか) を調べ、Response.Status に「401 Unauthorized 」エラー メッセージを設定します。この結果、IIS は NTLM または基本認証のいずれかを使用してユーザーの識別を試みるようになります。 
    • 




このスクリプト例を次に示します。



<% 



If Request("LOGON_USER") = "" Then



  Response.Status = "401 Unauthorized"



End if



%>



 



匿名ユーザーがアプリケーション内のすべてのファイルを利用できるようにする必要がある場合、Permission Checker コンポーネントは個々のユーザー アカウントを区別できません。ただし、このコンポーネントを使用して、指定された Web ページが存在するかどうかを確認し、匿名ユーザー アカウントがそのページへのアクセス権を持っているかどうかを調べることはできます。



重要



NTLM パスワード認証は、現在 Microsoft Internet Explorer でのみサポートされています。また、NTLM パスワード認証は、プロキシ サーバー経由では動作しない場合があります。このため、ユーザーが Internet Explorer 以外のブラウザまたはプロキシ サーバーを使用してサイトにアクセスしているときに、アプリケーションで匿名以外の認証が必要な場合は、基本認証とダイジェスト認証も有効にする必要があります。
