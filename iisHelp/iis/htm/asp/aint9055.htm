メタベースのセキュリティと信頼性

メタベースのアクセス制御

既定では、メタベース キーの値は、"metabase.bin" というディスク ファイルに格納されます。メタベースは、IIS 開始時にディスクからロードされ、IIS 終了時にディスクに格納されます。また、IIS 実行中に定期的に保存されます。このファイルは、不正に使用されないように保護する必要があります。このファイルは NTFS パーティションに格納し、Windows セキュリティを使用して保護することをお勧めします。

metabase.bin は、inetsrv ディレクトリに格納されます。このファイルは、移動や名前の変更が可能です。また、IIS が開始するときにこのファイルの場所を知らせる Windows レジストリの設定を変更できます。メタベース ファイルの場所または名前を変更するには、IIS を停止してから移動または名前の変更を行い、レジストリ キー LOCAL_MACHINE\SOFTWARE\Microsoft\INetMgr\Parameters を変更する必要があります。さらに、このキーに MetadataFile という REG_SZ 値を追加します。MetadataFile は、ドライブ名とファイル名を含む、メタベース ファイルの新しい完全なパスを指定します。

メタベースの信頼性

メタベースには、独自のカスタム バックアップ ポリシーを実装するか、または専用のバックアップ ツールを作成することができます。アプリケーションでは、IIsComputer オブジェクト (IIS Admin オブジェクトの場合)、または IMSAdminBase (IIS Admin Base オブジェクトの場合) が提供するメソッドを使って、メタベースのバックアップを管理できます。長期記憶域に複数のバックアップ バージョンを格納し、任意のバックアップ バージョンからメタベースを復元できます。また、バックアップ バージョンを列挙および削除することができます。

重要   メタベースのバックアップおよび復元機能は、IIS 5.1 で更新されています。レガシー メタベース バックアップは、バージョンの作成が目的であって、コンピュータ間での複製が目的ではありません。レガシー メタベースのバックアップは、そのバックアップを作成したコンピュータ上でのみ復元できます。メタベースのバックアップの作成および復元の詳細については、「メタベースのバックアップ」および「メタベースを復元する」を参照してください。

詳細については、Platform SDK の 「Advanced Programmatic Administration」を参照してください。
