Web アプリケーションの概要

オープン インターネット標準によって、分散コンピューティングのアーキテクチャは変化しました。Web の基本言語である HTML は、ユーザー インターフェイスの要素を表すための一般的な言語になりました。スクリプト言語と Java アプレットを複数のプラットフォームにまたがってサポートし、また埋め込み COM コンポーネントをサポートすることにより、ユーザーが対話的に使用するために、動的な要素を静的なテキストと組み合わせることが可能になりました。しかし Web テクノロジは、単にインターネット上で使用されるだけでなく、次に示すように、ビジネス上の用途にも適用されています。

  • コンポーネントベースの開発環境の作成
  • 企業内でのアプリケーションの配布
  • 新しいアプリケーションの迅速な作成とカスタマイズ
  • 通常の Web ブラウザを用いてリモートで実行するデータベースの更新
  • Web アプリケーションへのトランザクション処理の追加
  • ビジネス情報の共有
  • リソースの管理とリモートでのシステム管理

ここでは、Web アプリケーションの構築に役に立つ一般的な概念について説明します。ここで説明する項目は、次のとおりです。

  • 「クライアント/サーバー アーキテクチャでの構築」では、従来のクライアント/サーバー設計モデルについて簡単に説明します。
  • 「新しいシステム デザイン」では、クライアント/サーバー モデルの拡張機能としての、 n 階層システム設計および分散システム設計を紹介します。
  • 「Windows DNA」では、Web アプリケーション構築のための Microsoft のアーキテクチャである、Microsoft Windows DNA について概要を説明します。
  • 「IIS アーキテクチャ」では、IIS コンポーネントおよびプログラミング モデルについて説明します。
