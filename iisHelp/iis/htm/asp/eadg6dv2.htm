新しいシステム デザイン

現在のクライアント/サーバー アプリケーションは、従来のクライアント/サーバー アプリケーションとは異なるシステムで、"複数層アプリケーション" という新しい名前で呼ばれています。複数層アプリケーションは "n 層" アーキテクチャとも呼ばれます。このモデルでは、処理はクライアントとサーバーの間で分散され、ビジネス ロジックは中間の層に存在します。ほとんどのシステムでは次の 3 つの主要タスクが行われています。これらのタスクは、"n" 層モデルの各層に対応します。

タスク 説明
ユーザー インターフェイスと操作 次の図で "第 1 層" と表示されている層で、ユーザーが経験するもの全体を構成します。この層では、ユーザーがアプリケーションの操作、データ入力、要求に対する応答の表示を行うためのグラフィカル インターフェイスが提供されるだけではなく、データがクライアントに到着してからの、データ操作と書式設定も管理されます。Web アプリケーションでは、ブラウザがこの層のタスクを実行します。
ビジネス ロジック インターフェイス層とデータ サービス層の間の第 2 層は分散アプリケーション開発者の領域です。アプリケーションの処理を管理するルールを表すビジネス ロジックにより、ユーザーが、データに接続されます。このルールによって制御される機能は、日常業務をまねたもので、1 つのタスクであることも、また、複数の連続したタスクであることもあります。
データ サービス 下の図で第 3 層で表されているデータ サービスは、アプリケーション データへのアクセスを管理、提供する構造化データ ストア (SQL、Oracle データベース)、または非構造化データ ストア (Microsoft® Exchange、Microsoft® Message Queuing) から提供されています。1 つのアプリケーションで、1 つ以上のデータ ストアからのサービスを受けることができます。

3 層アーキテクチャでは、機能の主要な部分がそれぞれ分離されているので、プレゼンテーションは処理ルールやビジネス ロジックからは独立していて、データとも別になっています。このモデルでは、事前により多くの分析および設計を行う必要がありますが、維持にかかる経費を大きく削減できるので、長期的に見ると、機能的な柔軟性を高めることができます。次の図は、新しいシステム設計のさまざまな層をサービスする Microsoft のテクノロジを示しています。
