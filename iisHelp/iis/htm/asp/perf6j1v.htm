拡張性のある ASP アプリケーションを開発する

ASP の適切な使用

ASP を使用すると、C または C++ などの従来のサーバー側言語を使用した場合に比較して、Web アプリケーションを速く容易に作成できます。

ただし、ASP ベースのアプリケーションの作成が容易になる反面、そのアプリケーションに必要なサーバー処理とクライアント サーバー間の対話が複雑になります。拡張 ASP スクリプトで作成された Web アプリケーションの拡張性が十分得られない可能性があります。

拡張性の問題を避けるために、ASP を開発する際に考慮すべきポイントが 2 つあります。

  • ASP は、"接着剤" です。
  • ASP は、ビジネス ロジック用に使用するべきではありません。

最初のポイントは、ASP が HTML、DHTML を使用するクライアント側スクリプティング、および XML と綿密につながって、ユーザーと対話する強力なプラットフォームを作成することを強調しています。ASP スクリプティングは、ユーザー インターフェイスを Web アプリケーションのビジネス ロジックに結びつけるために設計されたもので、ASP はこのようなタスク用に最適化されました。

第 2 のポイントは、重要な注意事項です。ビジネス ロジックの大部分が ASP に埋め込まれていると、多くの場合、アプリケーションは適切な拡張性を持たなくなります。ASP がホストする ActiveX® スクリプト言語が大量のビジネス ロジック処理を実施できる能力があることは事実です。しかし、Web アプリケーションに必要なビジネス ロジックが通常のものを超えている場合は、ASP スクリプトに埋め込むのではなく、新しい COM コンポーネントにそのビジネス ロジックを組み入れることをお勧めします。

IIS での ASP の最適化

アプリケーションの設計に ASP の使用が適していることを確認し、ビジネス ロジックをまとめて COM コンポーネントにカプセル化した後で、Web アプリケーションのパフォーマンスと拡張性をさらに向上させる方法が 2 つあります。

  • ASP スクリプティングの最適化
  • IIS 構成の最適化

Web アプリケーションの設計の詳細については、「設計に関する決定事項」を参照してください。
