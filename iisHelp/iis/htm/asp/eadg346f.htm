IIS 要求処理

IIS は、HTTP 要求を受信すると、URL を評価し、要求されているものが静的コンテンツ (HTML) であるか、動的コンテンツ (ASP、またはISAPI) であるかを判断します。

要求処理アクション

要求 対策
HTML ページ IIS は即座に、ページを HTML 形式で返します。
ISAPI 拡張 これまでに実行されていなければ、IIS は ISAPI DLL をロードします。次に、Extension_Control_Block データ構造を介して、要求を拡張機能に送信します。
特定の ISAPI 拡張にマップされたファイル名の拡張子 IIS は、適切な DLL ファイルをロードし、Extension_Control_Block データ構造を介して要求を提示します。たとえば、.asp 拡張子を Asp.dll にマップすると、.asp 拡張子を持つファイルに対する要求はすべて Asp.dll に送られます。
CGI アプリケーション IIS は、新しいプロセスを作成し、クエリ文字列とそのほかのパラメータを提供します。これらのパラメータは、このプロセスの環境と標準入力 (STDIN) ハンドルを介した要求に含まれます。
