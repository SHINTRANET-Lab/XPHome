IIS とコンポーネント サービス

IIS とコンポーネント サービスを一緒に使用して、Web アプリケーションを構築するための基本アーキテクチャを形成します。IIS は、コンポーネント サービスが提供する機能を使用して、次のことを実行します。

  • アプリケーションを個別プロセスに分離します。
  • ASP 組み込みオブジェクトを含む、COM コンポーネント間の通信を管理します。
  • トランザクション ASP アプリケーションのためのトランザクション処理を調整します。

注   IIS Version 4.0 では、MTS (Microsoft Transaction Server) がトランザクション サポートを提供しました。IIS 5.1 および Windows XP では、コンポーネント サービスがコンポーネントの開発および配置のためのそのほかの多数の機能に加えて、MTS のトランザクション サポートをすべて提供します。

IIS は、Web アプリケーションをリソース ファイルの集合として定義し、これらのファイルは論理名前空間にグループ化されます。リソースをアプリケーションにグループ化することにより、名前空間内でデータを共有し、分離プロセスでアプリケーションを実行することが可能になります。

IIS の内部では、分離されたアプリケーションを Web Application Manager と呼ばれるオブジェクトを通して調整します。このオブジェクトには、パブリック インターフェイス IWAMAdmin が含まれ、これを使用して Web アプリケーションを管理するプログラムを作成できます。分離プロセスで Web アプリケーションを実行すると、IIS はコンポーネント サービスを使用してリソースに対する同時アクセスを調整し、COM コンポーネント間でコンテキスト情報を受け渡します。

IIS は、コンポーネント サービスの ObjectContext オブジェクトを使用して、ASP 組み込みオブジェクトに対する ASP アクセスで呼び出される COM コンポーネントを提供します。たとえば、Visual Basic で COM コンポーネントを構築し、そのコンポーネントで HTML ファイルからのフォームにアクセスする必要がある場合、コードは次のようになります。

Dim objObjectContext As ObjectContext
Dim vntIn As Variant 
Set objObjectContext = GetObjectContext ()
vntIn = objObjectContext.Item("Request").Form("Field1")

コンポーネント サービスは、ObjectContext を使用して、COM コンポーネントの特定のインスタンスに関する情報を維持します。IIS は、ASP スクリプトをコンパイルする際に、コンポーネント サービスを呼び出し、ASP スクリプトに関する情報を追跡するための ObjectContext を作成します。ObjectContext には、ASP スクリプトのインスタンスを一意に識別する ID プロパティが含まれます。たとえば、ASP スクリプトがコンポーネント サービスに登録されている COM コンポーネントのインスタンスを作成する場合、これらのオブジェクトは、それぞれ ASP スクリプトの ObjectContext に関連付けられます。同様に、@TRANSACTION ディレクティブ を使用したトランザクション ASP スクリプトを作成するときは、コンポーネント サービスに通知され、新しいトランザクションが開始されます。この後、コンポーネント サービスは、管理するあらゆるリソースの更新を調整します。コンポーネント サービスは、維持するリソースに加えられた変更をすべて追跡することにより、トランザクションの一貫性を保証します。

注   IIS Version 4.0 では、トランザクション サポートは MTS が行いました。IIS 5.1 および Windows XP では、コンポーネント サービスがほかの多数の機能に加えて、MTS のすべてのトランザクション サポートを行います。

アプリケーションの分離と管理の詳細については、「アプリケーションの境界を定義する」を参照してください。

トランザクション処理の詳細については、「トランザクション処理」、「トランザクションを理解する」、および Platform SDK にあるコンポーネント サービスのマニュアルを参照してください。
