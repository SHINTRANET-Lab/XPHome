Active Directory Service Interfaces (ADSI)

Microsoft Active Directory Service Interfaces (ADSI) は、COM ベースのディレクトリ サービス モデルで、ADSI に準拠するクライアント アプリケーションが、単一の標準インターフェイスを使用して Windows® ディレクトリ サービス、LDAP、NDS などの異なるディレクトリ プロトコルへ幅広くアクセスすることを可能にします。ADSI は、データ ストアやプロトコルの実装および動作からクライアント アプリケーションを保護します。

ADSI プロバイダと呼ばれるアプリケーションは、ADSI クライアント アプリケーションから利用できるようになっています。プロバイダが公開するデータは、プロバイダが定義したカスタム名前空間内で構成されます。プロバイダは、ADSI が定義するインターフェイスに加えて、ADSI スキーマも実装できます。このスキーマは、ADSI プロバイダが提供する名前空間構造およびオブジェクトに関するメタデータを提供するために使用されます。プロパティ属性は、プロパティの作成時にのみ設定してください。

ADSI の詳細については、Platform SDK の「Active Directory Service Interfaces Version 2.0 SDK」および「Active Directory Schema SDK」を参照してください。

ADSI と IIS

現在 IIS では、インターネット サイトの構成情報のほとんどを、IIS メタベースと呼ばれるカスタム データ ストアに格納しています。IIS では、アプリケーションがメタベースへアクセスして操作するための、下位レベルの DCOM インターフェイスを公開しています。また、メタベースへ容易にアクセスできるように、IIS は DCOM インターフェイスが提供する機能のほとんどをカバーする ADSI プロバイダを組み込み、これを ADSI 準拠のクライアント アプリケーションに公開しています。IIS で ADSI を使用する方法の詳細については、「IIS Admin オブジェクトを使用する」を参照してください。
