IIS のコア機能

IIS は、Web アプリケーションの構築に使用する基本的な機能を定義します。Active Server Pages (ASP) およびそのほかの Microsoft テクノロジは、この基本機能を拡張してアプリケーション開発のための機能を豊富に備えた環境を作り出しました。基本的なサーバー機能が ISAPI から公開されます。ISAPI の詳細については、Microsoft Platform SDK で説明されています。

IIS が提供するコア機能は、次のとおりです。

  • HTTP 接続の確立と維持
  • HTTP 要求の読み取りと HTTP 応答の書き込み
  • HTTP ヘッダーの変更
  • クライアント証明書情報の取得
  • 非同期接続の管理
  • URL (Uniform Resource Locator) の物理パスへのマッピング
  • アプリケーションの管理と実行
  • ファイルの転送

ASP は、COM アーキテクチャへのリンクを設定し、さらに Windows DNA の他の部分へのリンクを設定することにより、この機能を拡張します。同様に、ISAPI を使用して機能のカスタム セットを定義することにより、IIS アーキテクチャを拡張することができます。IIS のコア機能、ASP、および拡張されたアーキテクチャの関係を次の図に示します。
