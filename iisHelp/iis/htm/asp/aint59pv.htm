キー名とパス

メタベース キーは、メタベースにおける位置で、ファイル システムのディレクトリに相当します。メタベース パスは、スラッシュ (/) で区切られたキーの並びで、メタベースにおけるキーの位置を特定します。メタベースのキー名は、それぞれのメタベース パスで修飾されない限り一意ではありません。これは、同じ名前のファイルを異なるディレクトリに配置できることと同様です。

メタベース キーのパスと IIS Admin オブジェクトを使用して、そのキーに関連付けられた特定の IIS Admin オブジェクトにアクセスすることができます。IIS Admin オブジェクトは、ADsPath と呼ばれる Active Directory Service Interfaces (ADSI) のパスを使って、キーに関連付けられたオブジェクトを参照します。このパスは " IIS://" で始まり、次に「LocalHost」または特定のコンピュータ名を使用して、メタベースの最上位キーに関連付けられた IIsComputer オブジェクトを参照します。IIS Admin オブジェクトの詳細については、「IIS Admin オブジェクトを使用する」を参照してください。

ほかのメタベース キーに関連付けられているオブジェクトへのパスは、コンピュータ キーへの ADsPath を拡張したものです。たとえば、IIS インストール環境における 1 番目の Web サイトの ADsPath は、IIS://LocalHost/W3SVC/1 です。パス IIS://ComputerName/W3SVC/1 を使って、別のコンピュータ上のメタベースにアクセスすることもできます。

各 Web サイトは サーバー インスタンスであり、パスではその番号で参照されます。たとえば、"IIS://LocalHost/MSFTPSVC/3" は、3 番目の FTP サーバー インスタンスを表し、IIS://ComputerName/W3SVC/4 は、4 番目の Web サーバー インスタンスを表します。

各 Web サイトには、ルート仮想ディレクトリが関連付けられています。サーバー インスタンスに関連付けられているその他すべての仮想ディレクトリとディレクトリは、このルート仮想ディレクトリに従属します。ルート仮想ディレクトリの名前は、ROOT です。"IIS://LocalHost/MSFTPSVC/3/ROOT" は、ローカル コンピュータ上の 3 番目の FTP サーバーのルート仮想ディレクトリを表し、"IIS://LocalHost/W3SVC/4/ROOT" は、同じコンピュータ上の 4 番目の Web サーバーのルート仮想ディレクトリを表します。

サーバー インスタンスのルート仮想ディレクトリより下の構造を設定するには、仮想ディレクトリ、ディレクトリ、およびファイルを追加します。FTP サーバーの場合は、メタベース プロパティを仮想ディレクトリ レベルで設定できます。Web サーバーの場合は、プロパティをディレクトリおよびファイルのレベルで設定できます。たとえば、IIS://LocalHost/W3SVC/2/ROOT/MyVdir1/Dir1/File1 というパスを使って、読み取りアクセス権、書き込みアクセス権、実行アクセス権などのファイル プロパティを設定できます。
