Web サーバー作成ツール

概要

サーバーの作成、削除、停止、開始、および構成は、特に大規模なインターネットまたはイントラネットの Web サイトの管理者にとって、いずれも手間のかかる作業です。ここで提供するサンプルのツールでは、これらの作業の多くを自動化する方法を示します。

コードの説明

このサンプル スクリプトが提供する簡単なインターフェイスを通して、新しい Web サーバーを作成します。これを実現するには、IIS://LocalHost/W3SVC ノード上での GetObject の呼び出しから始まるいくつかの手順を行う必要があります。IIsWebService オブジェクトの ADSI メソッドである Create を使用して、新しい IIsWebServer オブジェクトを作成します。次にこのサーバー オブジェクトを構成し、SetInfo メソッドで新しい情報をメタベースに書き戻します。

スクリプトのこの段階では、サーバーはまだ完全に機能する Web サイトではありません。サーバーのノードはメタベースに作成されましたが、サーバーは動作していません。また、何かを格納するルート ディレクトリもありません。Web サイトを完成させるため、このツールでは ADSI オブジェクト IIsWebVirtualDir のインスタンスを作成し、このオブジェクトを新しいサーバーのルート ディレクトリとして構成します。ここで再び SetInfo を呼び出して、情報をメタベースに保存します。この段階まで問題がなければ、最後にサーバー オブジェクトの Start メソッドを呼び出します。

解説

このサンプルの主なステートメント群は、それぞれ独自にエラーコードをチェックしています。これは、一般にツールを作成する場合でも重要ですが、IIS メタベースを使用するツールにとっては特に重要です。

スクリプトの場所

このスクリプトは、インターネット サービス ディレクトリの ...\iissamples\sdk\admin\mkwebsrv.vbs に含まれています。
