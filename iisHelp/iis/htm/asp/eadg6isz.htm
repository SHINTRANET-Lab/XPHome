クライアント機能

作成するアプリケーションがさまざまなクライアント機能に対応する方法は、設計時の重要な決定事項の 1 つです。たとえば、ユーザーにとって最も重要な問題の 1 つに接続速度があります。アプリケーションでこの速度を決定することができれば、クライアント機能に合わせて、応答を調節することができます。クライアントからの要求の一部に、この情報が含まれている場合のみ、現在の接続速度をアプリケーションに認識させることができます。

クライアント機能に関する問題は、クライアント側、またはサーバー側のどちらかで解決できます。クライアント側のソリューションとしては、ダイナミック HTML (DHTML) を使用して、下の図にあるように、クライアントの現在の構成に関する記述を要求の一部として加えます。

この方法の利点は、次のとおりです。

  • クライアントとサーバー間の往復を減らすことができる
  • サーバーにかかる負荷を軽減できる
  • プロキシ サーバー キャッシュ技術により、アプリケーションをより敏感に反応させることができる

クライアント側でスクリプト機能がサポートされていない場合もあります。たとえば、インターネットに公開されているアプリケーションでは、クライアントでスクリプトがサポートされているかどうかを保証することはできません。つまり、一部のクライアントでは、このアプリケーションの実行は失敗に終わる可能性があります。また、セキュリティ上の理由から、スクリプトの実行が許可されていないネットワークにクライアントが存在する場合は、クライアント側からサーバー側リソースへはアクセスできません。

サーバー側では Browser Capabilities コンポーネントが使用されます。このコンポーネントにより要求に含まれている User Agent HTTP ヘッダーが読み込まれ、クライアントの機能が判別されます。IIS 3.0 および 4.0 に用意されていたバージョンの Browser Capabilities コンポーネントでは、クライアントの機能は静的リストから検索、確認されていました。次の図は、イベント シーケンスを示しています。

この方法では、リストが古くなった場合に、対処が難しくなり、アプリケーション設計者の負担となっていました。さらに重要なことに、この方法では、クライアントの機能にある構成可能な側面が考慮されていないので、要求が出されたときに実際に "有効" である設定を調べる方法が提供されていませんでした。

IIS 5.0 では、このような初期の設計上の制限を克服するために Browser Capabilities コンポーネントが改善されました。このコンポーネントは、クライアントの機能を表す cookie を返すクライアントからの個々の要求に合わせることができます。さらに、.asp ファイルに対する最初の要求に cookie が含まれていない場合は、クライアントでこれから実行されるスクリプトを呼び戻して、cookie を作成することができます。次の図は、イベント シーケンスを示しています。

このように Browser Capabilities コンポーネントが改善されたことにより、サーバー側ソリューションが 1 つ増えました。このテクニックでは、cookie が含まれていない要求がきたときに、特別のステータス コードを使用してクライアントへコール バックすることができます。このステータス コードを生成するには、.asp ファイルの先頭行に特別なメタ タグを記述します。たとえば、次のように指定します。

<!-- METADATA TYPE="Cookie" NAME="BrowsCap" SRC="sendcook.htm" -->

上の例は、IIS に対して、特別なステータス コード (HTTP 449 ステータス コード) を Internet Explorer 5 へ送信するように指示しています。続けて、このタグは、Internet Explorer 5 に対して、Sendcook.htm にあるスクリプトを実行するように指示しています。このスクリプトにより、クライアントの機能を表す cookie が生成されます。この cookie を受信したサーバーは、Browser Capabilirities コンポーネントとこの cookie を一緒に使用して、クライアントにどのように応答を返すかを決定します。

重要   Server.Transfer メソッド、または Server.Execute メソッドを使用したリダイレクトの結果として、クライアントにより要求されたファイルに METADATA メタ タグが存在する場合、このメタ タグは IIS では無視されます。ただし、実際にリダイレクトを含むファイルにある METADATA メタ タグは通常どおり処理されます。

この機能の使用方法の詳細については「cookie から Browser Capabilities を取得する」を参照してください。

DHTML クライアント機能の詳細については、MSDN Online の「DHTML reference information」を参照してください。新しい Browser Capabilities コンポーネント機能の例については、「Browser Capabilities コンポーネント」とPlatform SDK の IIS セクションにある ASP Samples または MSDN Online - Internet Information Services を参照してください。
