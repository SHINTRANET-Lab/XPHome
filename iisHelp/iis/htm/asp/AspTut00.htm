ASP チュートリアル

Microsoft® Active Server Pages (ASP) を使用すると、動的でインタラクティブな Web ページを作成することができます。ASP ページは、クライアントのブラウザに送る前に Web サーバーで処理するスクリプト コマンドを含む Hypertext Markup Language (HTML) ページです。これが、"サーバー側のスクリプト" と呼ばれる理由です。

HTML は、Web ページを作成するための最も簡単な言語ですが、静的な Web ページしか作成できません。Web クライアントが Web サーバーの静的 HTML ファイルを要求すると、Web サーバーは、計算を実行することなくクライアントに HTML ファイルを直接送信します (図 1)。その後、クライアントのブラウザは、ファイルの HTML コードを処理してコンテンツを表示します。

図 1

VBScript は、ASP ページを作成するための最も簡単な言語です。このチュートリアルのコード サンプルは、比較のために JScript で複製されたサンプル以外はすべて VBScript で作成されています。Web クライアントが Web サーバーの ASP ファイルを要求すると、Web サーバーは ASP エンジンを使用して ASP ファイルを送信します。ASP エンジンでは、すべてのサーバー側のスクリプト コードは実行されるか、HTML コードに変換されます (図 2)。変換されたコードは、Web クライアントに送信されます。

図 2

作成が困難な従来の Common Gateway Interface (CGI) アプリケーションとは異なり、ASP は Web アプリケーションの開発プロセスを大幅に簡素化するように設計されています。ASP では、スクリプトを数行追加するだけで、Web ページにデータベースの接続性や、高度なカスタマイズ機能を追加できます。以前は、このような機能を実現するのに PERL や C を習得する必要がありましたが、ASP では Microsoft JScript (ECMA 262 言語仕様を実装した Microsoft の言語)、Microsoft Visual Basic® Scripting Edition (VBScript)、または COM に準拠したスクリプト言語 (JavaScript や PERL など) のように一般的な Web スクリプト言語を使用できます。

ASP ページで Component Object Model (COM) コンポーネントを使用すると、ASP の機能を飛躍的に拡張することができます。COM コンポーネントは、ASP ページから呼び出すことができるコンパイル済みのコードです。COM コンポーネントは DLL としてコンパイルされる、セキュリティで保護されたコンパクトで再利用可能なオブジェクトです。Visual C++、Visual Basic、または COM をサポートするその他の言語で作成できます。

チュートリアルには、次のモジュールが含まれます。

  • モジュール 1 : ASP ページの作成   このモジュールでは、ASP ページを作成して実行し、データを収集するフォームを作成し、データベースに接続し、Microsoft Excel スプレッドシートを表示します。作業を始めるには、HTML ページの作成について多少の知識が必要です。
  • モジュール 2 : ASP ページでの COM コンポーネントの使用   このモジュールでは、Active Server Pages にインクルードする COM コンポーネントを使用して絵が切り替わる広告バナーを作成し、Web サイトのトラフィックを記録します。
  • モジュール 3 : Web アプリケーション内のセッションの状態を維持する   このモジュールでは、ASP のセッション状態の保持に使用できるさまざまなメソッドについて学習します。

始める前に

3 つのモジュールから構成されるこのチュートリアルでは、VBScript を使用した ASP ページの実装に役立つ具体的な手順およびサンプルを示します。チュートリアル サンプルを読み込んで実行するには、IIS が実行されているコンピュータの管理者権限を持っている必要があります。既定の IIS セキュリティ設定でチュートリアルを実行できますが、アクセス違反が発生した場合にはセキュリティ設定の変更が必要になる可能性があります。

ブラウザで ASP ページを開いたときにエラーが発生した場合は、次の操作を試みてください。

  • テキスト エディタで開かれている ASP ファイルを閉じます。サーバーは、テキスト エディタで開かれている HTML ページを表示できない場合があります。
  • 正しい名前と拡張子を持つファイルが x:\InetPub\Wwwroot\Tutorial に存在することを確認してください。
  • ブラウザのアドレス バーに入力した URL のスペルを確認し、必要に応じて修正します。URL は http://localhost/Tutorial/MyFile.asp または http://ComputerName/Tutorial/MyFile.asp のような形になっている必要があります。 
  • Web サーバーが稼動していることを確認します。アドレス バーに「http://localhost/localstart.asp」と入力することによって簡単に確認できます。"ページを表示できません" というエラーが返された場合、Web サービスが実行されていない可能性があります。Web サービスを起動するには、[スタート] ボタンをクリックし、[ファイル名を指定して実行] をクリックします。次に [名前] ボックスに「cmd」と入力します。コマンド ウィンドウで、「net start w3svc」と入力します。
問題が解決しない場合、発生しているエラーは一般的なエラーではありません。ブラウザに表示されるエラーを注意して読み、エラーの原因を判別できるかどうかを調べます。可能性のあるエラーとその説明を次に示します。
  • サーバー エラー   エラーの原因は IIS です。
  • VBScript または JScript エラー   おそらくコーディング エラーです。
  • ADODB   データベースに問題がある可能性があります。
  • セキュリティおよびアクセス拒否   これらのエラーは難解です。ファイルおよびフォルダのアクセス権、特にデータを変更する場合は、読み取り/書き込みアクセスを必要とするデータベース ファイルのアクセス権を確認する必要があります。

Web アクセス権は IIS の IIS スナップインで設定します。ただし、これらのアクセス権は、特定のファイルおよびフォルダの NTFS のアクセス権によって変更されることがあります。IIS で Web サイトへのユーザー アクセスを許可しても、NTFS ファイルのアクセス権が ASP ファイルやデータベース ファイルに対するアクセスを拒否する場合、Web ユーザーはそのファイルを表示できません。匿名認証を使用する場合、IWAM_ComputerName および IUSR_ComputerName アカウントへのアクセスを許可するように NTFS のアクセス権を設定する必要があります。セキュリティとアクセス権の設定については IIS のオンライン ヘルプを、NTFS セキュリティについては TechNet の IIS の記事を参照してください。

IIS のインストールをカスタマイズしていない場合は、既定のディレクトリとして x:\Inetpub\Wwwroot が作成されます。x は Windows がインストールされているドライブです。x:\Inetpub\Wwwroot で、Tutorial というディレクトリを作成します。各モジュールの演習では、作業を x:\Inetpub\Wwwroot\Tutorial ディレクトリに保存します。Web ブラウザで「http://localhost/Tutorial/file_name.asp (file_name.asp は表示するファイルの名前)」と入力し、ブラウザに作業結果を表示します。

Web ページにエラーがある場合、既定では Internet Explorer で一般的なメッセージを表示します。Internet Explorer で詳細なエラー メッセージを表示するには、IIS から送信されるエラーを回避する Internet Explorer の機能を無効にします。Internet Explorer で、[ツール] メニューをクリックし、[インターネット オプション] をクリックします。[詳細設定] タブをクリックし、[HTTP エラー メッセージを簡易表示する] チェック ボックスをオフにします。
