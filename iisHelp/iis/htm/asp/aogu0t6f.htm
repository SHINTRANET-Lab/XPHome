IIS Admin オブジェクトの概要

IIS Admin オブジェクトは、IIS メタベースに格納されている IIS 構成値を、ASP ページやカスタム アプリケーションから変更するために使用する COM オートメーション オブジェクトです。たとえば、ASP ページにスクリプトを記述することで、AccessRead、AccessWrite、および AccessExecute の各プロパティにアクセス権を設定できます。メタベースの詳細については、「IIS メタベースの紹介」を参照してください。

個々の IIS Admin オブジェクトは、メタベース内のキーに対応し、メタベースにおけるそのキーのパスによって参照されます。各オブジェクトはプロパティを持ち、各プロパティはメタベースに格納されます。IIS Admin オブジェクトを使って、IIS の構成を操作することができます。つまり、プロパティの変更、新しい仮想ディレクトリの追加、新しい Web サイトの追加などが可能です。

IIS Admin オブジェクトのプロパティを変更すると、対応するメタベース プロパティの値が変更されるため、ローカル コンピュータ (LM)、FTP サーバー (FTPSVC)、Web サーバー (W3SVC) などの、IIS の個々の要素を構成することができます。メタベースは、構造が階層的であり、プロパティ継承機能を備えているため、単一ファイルや単一サーバー、またはすべての Web サーバーやすべての FTP サーバーという単位でのプロパティの構成、またはその他多数のオブジェクト グループに共通のプロパティの構成が容易です。

ここでは、次の項目について説明します。

  • 「オブジェクト階層構造」では、IIS Admin オブジェクトとメタベースの関係、およびアプリケーションから IIS Admin オブジェクトへアクセスする方法について説明します。
  • 「ADSI の機能」では、IIS の管理に使用する ADSI の標準プロパティおよびメソッドを、IIS Admin オブジェクトがどのように実装するかを説明します。
