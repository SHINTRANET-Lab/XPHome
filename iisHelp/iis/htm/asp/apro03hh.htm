AuthPersistence

このプロパティは、接続要求全体に対する認証の継続性を指定します。このプロパティを設定するには、対応するサブフラグを TRUE に設定することをお勧めします。サブフラグを TRUE に設定すると、自動的に AuthPersistence にバイナリ値が設定されます。

AuthPersistSingleRequest が TRUE に設定されている場合、認証は単一の要求に対してのみ継続されます。AuthPersistSingleRequestIfProxy が TRUE に設定されている場合、要求がプロキシ要求で、その要求が Microsoft Proxy Server によって処理されていなければ、認証は単一の要求に対してのみ継続されます (つまり、Proxy Server と IIS がローカル コンピュータで動作中のときは、要求はリモート サーバーへ送られます)。AuthPersistSingleRequestAlwaysIfProxy が TRUE に設定されている場合、要求がプロキシによるものであれば、認証は単一の要求に対して有効です。

注   AuthPersistence のサブフラグは、一度に 1 つしか TRUE に設定できません。サブフラグは、AuthPersistSingleRequest、AuthPersistSingleRequestIfProxy、および AuthPersistSingleRequestAlwaysIfProxy です。これらの 3 つのプロパティのいずれかが TRUE に設定されると、ほかの 2 つのプロパティは自動的に FALSE に設定されます。AuthPersistence の値は、TRUE に設定されているサブフラグの値と必ず同じものになります。

データ型 長整数型
既定値 &H00000080 (または AuthPersistSingleRequestIfProxy=TRUE)

IIS Admin Base オブジェクトの追加情報

次の表は、IIS Admin Base オブジェクトを使用するコードで必要な追加情報の一覧です。

メタベース識別子 MD_AUTHORIZATION_PERSISTENCE
ユーザー タイプ IIS_MD_UT_FILE
既定のビットマスク設定 MD_AUTH_SINGLEREQUESTIFPROXY

ビットマスク値
定数 値 説明
MD_AUTH_SINGLEREQUEST 0x00000040 認証は単一の要求に対して有効です。IIS は要求の最後に認証をリセットし、次のセッション要求時に再認証を要求します。
MD_AUTH_SINGLEREQUESTIFPROXY 0x00000080 認証は単一の要求に対して有効です。現在認証されている要求がプロキシによるもので、IIS が MSPROXY を実行しているという特別のケースでない限り、IIS は要求の最後に認証をリセットします。
MD_AUTH_SINGLEREQUESTALWAYSIFPROXY 0x00000100 認証は単一の要求に対して有効です。現在認証されている要求が任意のタイプのプロキシによるものであれば、IIS は要求の最後に認証をリセットし、次のセッション要求時に再認証を要求します。
